## Learn（概念理解）

### PUT と PATCH の違い

| メソッド | 意味 | 未送信フィールド |
|---------|------|----------------|
| PUT | **全置換** | 未送信 = そのフィールドを空にする意図 |
| PATCH | **部分更新** | 未送信 = そのフィールドは変更しない |

```
# PUT: 全フィールド送信が前提
PUT /users/1 {"name": "Alice", "email": "alice@example.com"}

# PATCH: 変更したいフィールドだけ送信
PATCH /users/1 {"name": "Alice"}  ← email はそのまま
```

### 部分更新のバグパターン

```python
# ❌ Bad: PATCH なのに全フィールドを上書き（PUT と同じ実装）
@app.patch("/users/{user_id}")
def patch_user(user_id: int, user_data: UserPatch, db: Session = Depends(get_db)):
    user = db.query(User).filter(User.id == user_id).first()
    user.name = user_data.name    # name 未送信なら None が入る！
    user.email = user_data.email  # email 未送信なら None が入る！
    db.commit()
    return user
```

`UserPatch` の `name` と `email` は `Optional[str] = None` なので、送信しないと `None` になる。それをそのまま代入するとデータが消える。

### 正しい部分更新の実装

**`model_dump(exclude_unset=True)`** で「送信されたフィールドだけ」を取得する:

```python
# ✅ Good: exclude_unset=True で送信されたフィールドだけ更新
@app.patch("/users/{user_id}")
def patch_user(user_id: int, user_data: UserPatch, db: Session = Depends(get_db)):
    user = db.query(User).filter(User.id == user_id).first()
    if user is None:
        raise HTTPException(status_code=404, detail="ユーザーが見つかりません")

    update_data = user_data.model_dump(exclude_unset=True)
    for field, value in update_data.items():
        setattr(user, field, value)

    db.commit()
    db.refresh(user)
    return user
```

### `exclude_unset` vs `exclude_none` の違い

```python
data = UserPatch(name="Alice")  # email は送信されていない

data.model_dump()
# → {"name": "Alice", "email": None}

data.model_dump(exclude_unset=True)
# → {"name": "Alice"}  ← 送信されていないフィールドを除外

data.model_dump(exclude_none=True)
# → {"name": "Alice"}  ← None のフィールドを除外
```

**`exclude_unset` を使うべき理由**: ユーザーが明示的に `None` を送信した場合（値をクリアしたい場合）と、送信しなかった場合を区別できる。

## Practice（練習課題）

### 課題: 以下のコードのバグを見つけて修正せよ

```python
class ProductPatch(BaseModel):
    name: Optional[str] = None
    price: Optional[int] = None

@app.patch("/products/{product_id}")
def patch_product(product_id: int, data: ProductPatch, db: Session = Depends(get_db)):
    product = db.query(Product).filter(Product.id == product_id).first()
    product.name = data.name
    product.price = data.price
    db.commit()
    db.refresh(product)
    return product
```

**Q1.** `PATCH /products/1 {"name": "New Name"}` を送信すると何が起こるか？
**Q2.** なぜ `exclude_unset=True` が必要か、`exclude_none=True` ではダメな理由を説明せよ。
**Q3.** 修正コードを書け（存在チェックも追加）。

### 模範解答

<details>
<summary>クリックで表示</summary>

**A1.** `price` が `None` で上書きされ、商品の価格が消える。

**A2.** `exclude_none=True` だと、ユーザーが明示的に `{"price": null}` を送信した場合（価格をクリアしたい場合）も除外されてしまう。`exclude_unset=True` なら「送信されていない」と「明示的に null を送信した」を区別できる。

**A3.**
```python
from fastapi import HTTPException

@app.patch("/products/{product_id}")
def patch_product(product_id: int, data: ProductPatch, db: Session = Depends(get_db)):
    product = db.query(Product).filter(Product.id == product_id).first()
    if product is None:
        raise HTTPException(status_code=404, detail="商品が見つかりません")

    update_data = data.model_dump(exclude_unset=True)
    for field, value in update_data.items():
        setattr(product, field, value)

    db.commit()
    db.refresh(product)
    return product
```

### よくある間違い

- `exclude_none=True` を使う（明示的に null を送信した場合との区別ができない）
- `if user_data.name is not None:` で各フィールドを個別チェック（フィールドが増えると破綻する）

- PUT の実装も同じように変えてしまう（PUT は全置換なので現状のままでよい）
