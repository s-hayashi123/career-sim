## Learn（概念理解）

### None チェック忘れは最も多いバグの1つ

DB からレコードを取得するとき、**見つからなければ `None` が返る**。`None` に対して属性アクセスすると `AttributeError` → 500 になる:

```python
# ❌ Bad: None チェックなし → AttributeError で 500
user = db.query(User).filter(User.id == user_id).first()
print(f"Found user: {user.name}")  # user が None なら AttributeError!
return user

# ✅ Good: None チェック（Guard Clause）
user = db.query(User).filter(User.id == user_id).first()
if user is None:
    raise HTTPException(status_code=404, detail="ユーザーが見つかりません")
return user
```

### Guard Clause パターン

**異常系を先に return/raise して、正常系のインデントを浅くする**書き方:

```python
# ❌ Bad: ネストが深い
def get_user(user_id: int, db: Session):
    user = db.query(User).filter(User.id == user_id).first()
    if user is not None:
        if user.is_active:
            return user
        else:
            raise HTTPException(status_code=403, detail="無効なユーザー")
    else:
        raise HTTPException(status_code=404, detail="見つかりません")

# ✅ Good: Guard Clause で早期 return
def get_user(user_id: int, db: Session):
    user = db.query(User).filter(User.id == user_id).first()
    if user is None:
        raise HTTPException(status_code=404, detail="見つかりません")
    if not user.is_active:
        raise HTTPException(status_code=403, detail="無効なユーザー")
    return user
```

### HTTPException と 404

FastAPI の `HTTPException` を raise すると、自動的に適切な JSON レスポンスが返る:

```python
from fastapi import HTTPException

raise HTTPException(status_code=404, detail="ユーザーが見つかりません")
```

レスポンス:
```json
{"detail": "ユーザーが見つかりません"}
```

### 404 Not Found の正しい使い方

| 状況 | ステータスコード | 理由 |
|------|---------------|------|
| `/users/999` でユーザーが存在しない | 404 | リソースが見つからない |
| `/users/abc` で ID の形式が不正 | 422 | バリデーションエラー |
| DB 接続エラー | 500 | サーバー内部エラー |

**500 は「サーバー側のバグ」**。存在しないリソースへのアクセスは正常な操作であり、404 で返すべき。

## Practice（練習課題）

### 課題: 以下のコードのバグを見つけて修正せよ

```python
@app.get("/orders/{order_id}")
def get_order(order_id: int, db: Session = Depends(get_db)):
    order = db.query(Order).filter(Order.id == order_id).first()
    total = order.quantity * order.price
    return {"order": order, "total": total}
```

**Q1.** このコードにはどんなバグがあるか？
**Q2.** `GET /orders/99999` を呼んだとき何が起こるか？
**Q3.** Guard Clause パターンで修正せよ。

### 模範解答

<details>
<summary>クリックで表示</summary>

**A1.** `order` が `None`（存在しない `order_id`）のとき、`order.quantity` で `AttributeError` が発生し 500 エラーになる。

**A2.** `AttributeError: 'NoneType' object has no attribute 'quantity'` が発生し、500 Internal Server Error が返る。

**A3.**
```python
from fastapi import HTTPException

@app.get("/orders/{order_id}")
def get_order(order_id: int, db: Session = Depends(get_db)):
    order = db.query(Order).filter(Order.id == order_id).first()
    if order is None:
        raise HTTPException(status_code=404, detail="注文が見つかりません")
    total = order.quantity * order.price
    return {"order": order, "total": total}
```

</details>
