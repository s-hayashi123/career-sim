# P02-10: スプリントレトロスペクティブ

> **シナリオ**: Project 02 のスプリント完了、振り返りを行う
> **想定時間**: 3h
> **スキルタグ**: P8（ドキュメント・プロセス改善）

---

## Learn（概念理解）

### 2回目のレトロで意識すること

Project 01 の振り返りとの違い:
- **前回の Try を振り返る** — 「次に試す」と書いたことを実際に試せたか？
- **成長の比較** — P01 のときと比べて何が変わったか？
- **パターンの発見** — 繰り返し発生する問題はないか？

### レトロスペクティブの深め方

```markdown
# ❌ 浅い振り返り
## Good
- バグ修正できた

## Bad
- 時間がかかった

# ✅ 深い振り返り
## Good
- P02-01 で「DB 制約はアプリ側チェックだけでは不十分」と学び、以降の
  シナリオでも制約ファーストで考えるようになった
- P02-04 のグローバル例外ハンドラ実装で、セキュリティ視点でコードを見る習慣がついた
- ADR を書くことで「なぜそうしたか」を言語化する練習になった

## Bad
- P02-05 の exclude_unset と exclude_none の違いの理解に時間がかかった
  → Pydantic のドキュメントをもっと早く読むべきだった
- テスト整理（P02-06）で conftest.py のスコープを間違えて、テスト間で
  DB 状態が汚染された
```

## Practice（練習課題）

### 課題: 以下の振り返りの改善点を指摘せよ

```markdown
## P01 の Try の振り返り
（空欄）

## Good
- バグをたくさん直した
- テストを書いた
- ADR を書いた

## Bad
- 特になし

## Try
- 引き続き頑張る
```

### 模範解答

<details>
<summary>クリックで表示</summary>

**問題点:**
1. **P01 の Try を振り返っていない** — 前回「次に試す」と決めたことの振り返りがない
2. **Good が抽象的** — 何をどう直した/書いたか不明
3. **Bad が「特になし」** — 振り返りの意味がない。必ず改善点はある
4. **Try が「頑張る」** — 具体的なアクションでない

**改善:**
```markdown
## P01 の Try の振り返り
- 「バグ修正はエラーメッセージから該当行を特定する」→ ✅ P02-02 で実践できた
- 「Conventional Commits のチートシートをブックマーク」→ ❌ やっていなかった

## Good
- P02-01 でレースコンディションの概念を学び、DB 制約の重要性を体感した
- P02-04 でセキュリティ観点（スタックトレース漏洩）を初めて意識した
- テスト整理で conftest.py を使いこなせるようになった

## Bad
- P02-05 で Pydantic の exclude_unset を知らず、独自実装しようとして1時間ロスした
- ADR の書き方が毎回迷う（テンプレートを用意すべきだった）

## Try
- 実装前に公式ドキュメントの該当セクションを5分読む習慣をつける
- ADR テンプレートを `docs/adrs/TEMPLATE.md` として用意する
```

</details>

## Mission（本番課題）

### 状況

Project 02 の全シナリオ（P02-01 〜 P02-09）が完了しました。
スプリントの終わりとして、振り返りとスコアカードを作成します。

### やること

1. **P01 の Try を振り返る**
   - `career-sim/docs/retros/P01-retro.md` を読む
   - 「次に試す」と書いたことを実際に試せたか振り返る

2. **レトロスペクティブを作成する**
   - ファイル: `career-sim/docs/retros/P02-retro.md`
   - 以下を含める:
     - P01 の Try の振り返り（✅/❌ で達成状況）
     - P02-01 〜 P02-09 で何をやったか（各1行サマリー）
     - Good: 具体的に（最低3つ）
     - Bad: 具体的に（最低3つ）
     - Try: 具体的なアクション（最低3つ）
     - 学んだ技術の一覧
     - P01 との比較（成長した点）

3. **スコアカードを作成する**
   - ファイル: `career-sim/scorecards/P02-scorecard.md`
   - 各シナリオの Review チェックリストに基づく達成率
   - Issue/PR 番号の記録

4. **evidence-index を更新する**
   - ファイル: `career-sim/docs/evidence-index.md`
   - Project 02 で作成した証跡（PR、ADR）を追加

5. **Project 02 全体の達成度を算出する**
   - 必須項目の達成率
   - 加点項目の達成数

### P02 で学んだ技術の例

- UNIQUE 制約と IntegrityError
- 404 ハンドリングと Guard Clause
- 更新時ユニーク検証（自分自身除外パターン）
- グローバル例外ハンドラ
- PATCH の部分更新（`exclude_unset`）
- テストの unit/integration 分離
- Pydantic `extra="forbid"`
- 外部キー制約と ON DELETE
- Alembic マイグレーションの冪等性

### 加点項目

- [ ] P01 との比較で成長した点が書かれている
- [ ] 学んだ技術が具体的に列挙されている
- [ ] Bad に対する改善アクションが Try に対応している

### よくある間違い

- P01 の Try を振り返らない（前回の改善提案を無視している）
- Good/Bad/Try が P01 と同じ粒度で深まっていない（2回目はより具体的であるべき）
- スコアカードに Issue/PR 番号を書き忘れる
- evidence-index を更新しない
