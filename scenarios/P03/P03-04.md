## Learn（概念理解）

### クエリパラメータによるフィルタリング

REST API では **クエリパラメータ** で検索条件を指定する:

```
GET /tasks?status=TODO&assignee_id=1&priority=HIGH
```

FastAPI での実装:

```python
from fastapi import Query

@app.get("/tasks")
def list_tasks(
    status: TaskStatus | None = None,
    assignee_id: int | None = None,
    priority: TaskPriority | None = None,
    db: Session = Depends(get_db)
):
    query = db.query(Task)

    if status:
        query = query.filter(Task.status == status)
    if assignee_id:
        query = query.filter(Task.assignee_id == assignee_id)
    if priority:
        query = query.filter(Task.priority == priority)

    return query.all()
```

### SQLAlchemy の filter vs filter_by

| メソッド | 使い方 | 用途 |
|---------|--------|------|
| `filter()` | `filter(Task.status == "TODO")` | 複雑な条件（比較演算子、OR、LIKE） |
| `filter_by()` | `filter_by(status="TODO")` | シンプルな等価条件 |

```python
# どちらも同じ結果
query.filter(Task.status == "TODO")
query.filter_by(status="TODO")

# filter() のみ可能
query.filter(Task.created_at > datetime(2024, 1, 1))
query.filter(or_(Task.status == "TODO", Task.status == "IN_PROGRESS"))
```

### ページネーション（limit / offset）

大量のデータを **ページ分割** して返す:

```
GET /tasks?limit=10&offset=0   → 1-10件目
GET /tasks?limit=10&offset=10  → 11-20件目
GET /tasks?limit=10&offset=20  → 21-30件目
```

実装:

```python
@app.get("/tasks")
def list_tasks(
    limit: int = Query(default=100, le=100),  # 最大100件
    offset: int = Query(default=0, ge=0),     # 0以上
    db: Session = Depends(get_db)
):
    tasks = db.query(Task).limit(limit).offset(offset).all()
    return tasks
```

**ベストプラクティス**:
- `limit` のデフォルト値を設定（例: 100）
- `limit` の最大値を制限（例: 1000）
- `offset` は 0 以上

### 複数条件の組み合わせ

```python
@app.get("/tasks")
def list_tasks(
    status: TaskStatus | None = None,
    assignee_id: int | None = None,
    priority: TaskPriority | None = None,
    limit: int = Query(default=100, le=100),
    offset: int = Query(default=0, ge=0),
    db: Session = Depends(get_db)
):
    query = db.query(Task)

    # フィルタリング
    if status:
        query = query.filter(Task.status == status)
    if assignee_id:
        query = query.filter(Task.assignee_id == assignee_id)
    if priority:
        query = query.filter(Task.priority == priority)

    # ソート（最新順）
    query = query.order_by(Task.created_at.desc())

    # ページネーション
    tasks = query.limit(limit).offset(offset).all()

    return tasks
```

### ページネーションのレスポンス設計

単純にリストを返すだけでなく、**総件数や次のページの有無**を含めると便利:

```python
# ❌ シンプルだが情報不足
{
  "tasks": [...]
}

# ✅ ページネーション情報付き
{
  "items": [...],
  "total": 150,
  "limit": 10,
  "offset": 0,
  "has_next": true
}
```

実装例:

```python
class PaginatedTasksResponse(BaseModel):
    items: list[TaskResponse]
    total: int
    limit: int
    offset: int
    has_next: bool

@app.get("/tasks", response_model=PaginatedTasksResponse)
def list_tasks(limit: int = 100, offset: int = 0, db: Session = Depends(get_db)):
    query = db.query(Task)
    total = query.count()
    tasks = query.limit(limit).offset(offset).all()

    return {
        "items": tasks,
        "total": total,
        "limit": limit,
        "offset": offset,
        "has_next": offset + limit < total
    }
```

## Practice（練習課題）

### 課題: 以下のコードの問題点を見つけて修正せよ

```python
@app.get("/products")
def list_products(category: str, db: Session = Depends(get_db)):
    products = db.query(Product).filter(Product.category == category).all()
    return products
```

**Q1.** category が指定されなかった場合、何が起こるか？
**Q2.** ページネーションがないことによる問題は何か？
**Q3.** 修正したコードを書け（フィルタ任意、ページネーション付き）。

### 模範解答

<details>
<summary>クリックで表示</summary>

**A1.** `category` が必須パラメータなので、指定されないと 422 Unprocessable Entity エラーになる。

**A2.** 商品が10万件ある場合、全件返そうとしてメモリ不足やタイムアウトが発生する。

**A3. 修正コード**:

```python
@app.get("/products")
def list_products(
    category: str | None = None,
    limit: int = Query(default=100, le=1000),
    offset: int = Query(default=0, ge=0),
    db: Session = Depends(get_db)
):
    query = db.query(Product)

    # category は任意
    if category:
        query = query.filter(Product.category == category)

    # ページネーション
    products = query.limit(limit).offset(offset).all()

    return products
```

</details>

<details>
<summary>複数条件のフィルタリング実装例</summary>

```python
@app.get("/tasks")
def list_tasks(
    status: TaskStatus | None = None,
    assignee_id: int | None = None,
    priority: TaskPriority | None = None,
    limit: int = Query(default=100, le=100),
    offset: int = Query(default=0, ge=0),
    db: Session = Depends(get_db)
):
    query = db.query(Task)

    # フィルタリング（全て任意）
    if status:
        query = query.filter(Task.status == status)
    if assignee_id is not None:  # 0 も有効な ID なので is not None
        query = query.filter(Task.assignee_id == assignee_id)
    if priority:
        query = query.filter(Task.priority == priority)

    # ソート
    query = query.order_by(Task.created_at.desc())

    # 総件数
    total = query.count()

    # ページネーション
    tasks = query.limit(limit).offset(offset).all()

    return {
        "items": tasks,
        "total": total,
        "limit": limit,
        "offset": offset
    }
```

## Review（自己採点チェックリスト）

### 必須項目

- [ ] GET /tasks に status, assignee_id, priority のクエリパラメータがある
- [ ] 全てのフィルタが任意（指定されなければ全件）
- [ ] limit と offset でページネーションができる
- [ ] created_at の降順でソートされている
- [ ] レスポンスに total（総件数）が含まれている
- [ ] フィルタなしのテストがある
- [ ] 単一条件（status のみ）のテストがある
- [ ] 複数条件（status & priority）のテストがある
- [ ] ページネーション（limit & offset）のテストがある
- [ ] 全テストが通る

### 加点項目

- [ ] limit のデフォルト値と最大値が設定されている
- [ ] offset が 0 未満の場合にバリデーションエラーが出る
- [ ] レスポンスに has_next（次のページがあるか）が含まれている
- [ ] project_id でもフィルタできる
- [ ] テストで境界値（limit=1, offset=0）を確認している

### よくある間違い

- assignee_id を `if assignee_id:` でチェックする（0 が有効な ID の場合に除外されてしまう。`is not None` を使う）
- limit のデフォルト値がない（全件返してメモリ不足）
- フィルタが AND ではなく OR になっている（複数条件が正しく機能しない）
- ソート順が指定されていない（結果が不定）
- total を計算せず、len(items) を返している（ページネーション後の件数になる）
- limit > 総件数の場合のテストがない
