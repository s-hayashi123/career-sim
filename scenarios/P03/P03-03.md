## Learn（概念理解）

### 外部キー（Foreign Key）とは

**別のテーブルの主キーを参照する** カラム。データの整合性を保証する:

```python
# ❌ Bad: 外部キー制約なし
class Task(Base):
    __tablename__ = "tasks"
    id = Column(Integer, primary_key=True)
    project_id = Column(Integer)  # ただの Integer
    title = Column(String(255))

# 問題: 存在しない project_id を指定しても DB が受け入れてしまう
# task.project_id = 999 → DB に INSERT 成功（でも projects.id=999 は存在しない）

# ✅ Good: 外部キー制約あり
class Task(Base):
    __tablename__ = "tasks"
    id = Column(Integer, primary_key=True)
    project_id = Column(Integer, ForeignKey("projects.id"), nullable=False)
    title = Column(String(255), nullable=False)

# 存在しない project_id を指定すると IntegrityError
```

### SQLAlchemy の relationship

**モデル間の関連を定義する** ことで、Python コードから簡単にリレーションを扱える:

```python
# Project モデル
class Project(Base):
    __tablename__ = "projects"
    id = Column(Integer, primary_key=True)
    name = Column(String(255), nullable=False, unique=True)

    # 1対多: 1つの Project に複数の Task
    tasks = relationship("Task", back_populates="project", cascade="all, delete-orphan")

# Task モデル
class Task(Base):
    __tablename__ = "tasks"
    id = Column(Integer, primary_key=True)
    project_id = Column(Integer, ForeignKey("projects.id"), nullable=False)
    title = Column(String(255), nullable=False)

    # 多対1: 1つの Task は1つの Project に属する
    project = relationship("Project", back_populates="tasks")

# 使い方
project = db.query(Project).first()
print(project.tasks)  # このプロジェクトに紐づくタスクのリスト

task = db.query(Task).first()
print(task.project.name)  # このタスクが属するプロジェクトの名前
```

### cascade="all, delete-orphan" の意味

| オプション | 挙動 |
|-----------|------|
| `cascade="all"` | 親（Project）が削除されると、子（Task）も自動削除 |
| `delete-orphan` | 親から切り離された子も削除 |

```python
# cascade がないと
project = db.query(Project).first()
db.delete(project)
db.commit()
# → tasks テーブルに project_id が NULL または孤立したレコードが残る（データ不整合）

# cascade="all, delete-orphan" があると
db.delete(project)
db.commit()
# → 紐づく tasks も自動削除される
```

### Pydantic の入れ子レスポンス

関連データを含むレスポンスを返す:

```python
# TaskResponse にプロジェクト情報を含める
class ProjectResponse(BaseModel):
    id: int
    name: str

    class Config:
        from_attributes = True

class TaskResponse(BaseModel):
    id: int
    title: str
    description: str | None
    status: str
    project: ProjectResponse  # 入れ子

    class Config:
        from_attributes = True

# API レスポンス例
{
  "id": 1,
  "title": "タスク1",
  "description": "説明",
  "status": "TODO",
  "project": {
    "id": 1,
    "name": "Project A"
  }
}
```

### ENUM 型の実装

ステータスや優先度を **固定値に制限する**:

```python
from enum import Enum

# Python Enum
class TaskStatus(str, Enum):
    TODO = "TODO"
    IN_PROGRESS = "IN_PROGRESS"
    DONE = "DONE"

class TaskPriority(str, Enum):
    LOW = "LOW"
    MEDIUM = "MEDIUM"
    HIGH = "HIGH"

# SQLAlchemy モデル
class Task(Base):
    __tablename__ = "tasks"
    # ...
    status = Column(Enum(TaskStatus), nullable=False, default=TaskStatus.TODO)
    priority = Column(Enum(TaskPriority), nullable=False, default=TaskPriority.MEDIUM)

# Pydantic スキーマ
class TaskCreate(BaseModel):
    title: str
    description: str | None = None
    status: TaskStatus = TaskStatus.TODO  # デフォルト値
    priority: TaskPriority = TaskPriority.MEDIUM
```

## Practice（練習課題）

### 課題: 以下のコードの問題点を見つけて修正せよ

```python
# Order モデル
class Order(Base):
    __tablename__ = "orders"
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer)
    total = Column(Float)

# OrderItem モデル
class OrderItem(Base):
    __tablename__ = "order_items"
    id = Column(Integer, primary_key=True)
    order_id = Column(Integer)
    product_name = Column(String(255))
    quantity = Column(Integer)
```

**Q1.** 外部キー制約がないことによるリスクを2つ挙げよ。
**Q2.** relationship を追加して、`order.items` でアイテム一覧を取得できるようにせよ。
**Q3.** Order 削除時に OrderItem も削除するには、どの cascade オプションを使うか？

### 模範解答

<details>
<summary>クリックで表示</summary>

**A1. リスク**:
1. 存在しない `order_id` を持つ `OrderItem` が作成できる（データ不整合）
2. `Order` を削除しても `OrderItem` が残り、孤立レコードになる

**A2. 修正コード**:

```python
class Order(Base):
    __tablename__ = "orders"
    id = Column(Integer, primary_key=True)
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False)
    total = Column(Float, nullable=False)

    # 1対多
    items = relationship("OrderItem", back_populates="order", cascade="all, delete-orphan")

class OrderItem(Base):
    __tablename__ = "order_items"
    id = Column(Integer, primary_key=True)
    order_id = Column(Integer, ForeignKey("orders.id"), nullable=False)
    product_name = Column(String(255), nullable=False)
    quantity = Column(Integer, nullable=False)

    # 多対1
    order = relationship("Order", back_populates="items")
```

**A3.** `cascade="all, delete-orphan"`

</details>

@app.post("/tasks", status_code=201)
def create_task(task: TaskCreate, db: Session = Depends(get_db)):
    # project_id の存在確認
    project = db.query(Project).filter(Project.id == task.project_id).first()
    if not project:
        raise HTTPException(status_code=422, detail="指定されたプロジェクトが存在しません")

    db.add(db_task)
    db.commit()
    db.refresh(db_task)

@app.get("/tasks/{task_id}", response_model=TaskResponse)
def get_task(task_id: int, db: Session = Depends(get_db)):
    task = db.query(Task).filter(Task.id == task_id).first()

        raise HTTPException(status_code=404, detail="Task not found")

def test_delete_project_cascades_to_tasks(client, db):
    # プロジェクト作成
    project_response = client.post("/projects", json={"name": "Project A"})
    project_id = project_response.json()["id"]

    # タスク作成
    task_response = client.post("/tasks", json={
        "project_id": project_id,
        "title": "Task 1"
    })
    task_id = task_response.json()["id"]

    client.delete(f"/projects/{project_id}")

    response = client.get(f"/tasks/{task_id}")

## Review（自己採点チェックリスト）

### 必須項目

- [ ] Task モデルが定義されている
- [ ] `project_id` に外部キー制約がある（`ForeignKey("projects.id")`）
- [ ] TaskStatus と TaskPriority の Enum が定義されている
- [ ] Project モデルに `tasks` relationship がある
- [ ] `cascade="all, delete-orphan"` が設定されている
- [ ] Alembic マイグレーションが作成・適用されている
- [ ] POST /tasks が実装され、201 を返す
- [ ] GET /tasks/{id} のレスポンスにプロジェクト情報が含まれる
- [ ] 存在しない project_id で 422 を返す
- [ ] プロジェクト削除時に紐づくタスクも削除されることをテストしている
- [ ] 全テストが通る

### 加点項目

- [ ] assignee_id にも外部キー制約がある
- [ ] デフォルト値（status=TODO, priority=MEDIUM）が設定されている
- [ ] GET /tasks/{id} で assignee 情報も入れ子で返している
- [ ] Enum の不正な値（例: "INVALID"）で 422 を返すテストがある
- [ ] マイグレーションに downgrade が定義されている

### よくある間違い

- 外部キー制約を追加しない（`Column(Integer)` だけ）
- relationship を定義しない（手動で JOIN を書く必要が出る）
- cascade を設定せず、プロジェクト削除時にタスクが孤立する
- 存在しない project_id のバリデーションがない（500 エラーになる）
- 入れ子レスポンスの `from_attributes=True` を忘れる（Pydantic が変換できない）
- POST で 200 を返している（201 を返すべき）
