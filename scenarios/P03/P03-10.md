### OpenAPI / Swagger とは

**API の仕様を記述する標準フォーマット**。FastAPI は自動生成するが、**説明を追加する**ことで実用的になる:

```python
# ❌ Bad: 説明なし
@app.post("/tasks")
def create_task(task: TaskCreate, db: Session = Depends(get_db)):
    ...

# ✅ Good: 説明付き
@app.post(
    "/tasks",
    status_code=201,
    response_model=TaskResponse,
    summary="タスクを作成",
    description="新しいタスクを作成します。project_id は必須で、存在するプロジェクトを指定してください。"
)
def create_task(task: TaskCreate, db: Session = Depends(get_db)):
    """
    タスク作成エンドポイント

    - **project_id**: タスクが属するプロジェクトの ID（必須）
    - **title**: タスクのタイトル（必須）
    - **status**: タスクのステータス（デフォルト: TODO）
    - **priority**: タスクの優先度（デフォルト: MEDIUM）
    """
    ...
```

**Swagger UI で見ると**:
- `summary` がエンドポイントの一覧に表示される
- `description` がエンドポイント詳細に表示される
- docstring が "Description" セクションに表示される

### Pydantic スキーマの説明

```python
from pydantic import BaseModel, Field

# ❌ Bad: フィールドの説明なし
class TaskCreate(BaseModel):
    project_id: int
    title: str
    status: TaskStatus = TaskStatus.TODO

# ✅ Good: Field で説明を追加
class TaskCreate(BaseModel):
    project_id: int = Field(..., description="タスクが属するプロジェクトの ID", example=1)
    title: str = Field(..., description="タスクのタイトル", min_length=1, max_length=255, example="API 実装")
    status: TaskStatus = Field(default=TaskStatus.TODO, description="タスクのステータス")
    priority: TaskPriority = Field(default=TaskPriority.MEDIUM, description="タスクの優先度")

    class Config:
        schema_extra = {
            "example": {
                "project_id": 1,
                "title": "ユーザー認証機能の実装",
                "status": "TODO",
                "priority": "HIGH"
            }
        }
```

## Practice（練習課題）

### 課題: 以下のエンドポイントに OpenAPI の説明を追加せよ

```python
@app.get("/products/{product_id}")
def get_product(product_id: int, db: Session = Depends(get_db)):
    product = db.query(Product).filter(Product.id == product_id).first()
    if not product:
        raise HTTPException(status_code=404, detail="Product not found")
    return product
```

**Q1.** `summary` と `description` を追加せよ。
**Q2.** docstring を書け（パラメータの説明を含む）。
**Q3.** 404 のレスポンスを `responses` パラメータで定義せよ。

### 模範解答

<details>
<summary>クリックで表示</summary>

```python
@app.get(
    "/products/{product_id}",
    response_model=ProductResponse,
    summary="商品を取得",
    description="指定された ID の商品情報を取得します。",
    responses={
        200: {"description": "商品情報を返します"},
        404: {"description": "指定された ID の商品が見つかりません"}
    }
)
def get_product(product_id: int, db: Session = Depends(get_db)):
    """
    商品情報取得エンドポイント

    - **product_id**: 取得したい商品の ID
    """
    product = db.query(Product).filter(Product.id == product_id).first()
    if not product:
        raise HTTPException(status_code=404, detail="Product not found")
    return product
```
