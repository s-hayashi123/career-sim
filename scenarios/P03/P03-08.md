## Learn（概念理解）

### 監査ログ（Audit Log）とは

**データの変更履歴を記録する**仕組み。実務では以下の用途で必須:

- **トラブルシューティング**: 「誰がいつ何を変更したか」を追跡
- **コンプライアンス**: 法規制対応（GDPR、SOX など）
- **セキュリティ**: 不正操作の検出

### 監査ログの設計パターン

| パターン | 実装方法 | メリット | デメリット |
|---------|---------|---------|----------|
| **DB トリガー** | DB レベルで自動記録 | アプリケーションに依存しない | DB 依存、テストが難しい |
| **アプリケーション層** | 更新処理で明示的に記録 | 柔軟、テストしやすい | 記録漏れのリスク |
| **イベントソーシング** | 全ての変更をイベントとして記録 | 完全な履歴 | 複雑、オーバーヘッド大 |

**このシナリオでは「アプリケーション層」を採用**。

### 変更履歴テーブルの設計

```python
class TaskActivity(Base):
    __tablename__ = "task_activities"

    id = Column(Integer, primary_key=True, index=True)
    task_id = Column(Integer, ForeignKey("tasks.id"), nullable=False)
    field_name = Column(String(255), nullable=False)  # 変更されたフィールド
    old_value = Column(Text, nullable=True)           # 変更前の値
    new_value = Column(Text, nullable=True)           # 変更後の値
    changed_by = Column(Integer, ForeignKey("users.id"), nullable=True)  # 変更者
    changed_at = Column(DateTime, nullable=False, server_default=func.now())

    # リレーション
    task = relationship("Task", back_populates="activities")
```

**記録する情報**:
- **task_id**: どのタスクの変更か
- **field_name**: どのフィールドが変更されたか（status, priority など）
- **old_value / new_value**: 変更前後の値
- **changed_by**: 誰が変更したか（認証があれば）
- **changed_at**: いつ変更されたか

### 変更差分の検出と記録

```python
@app.put("/tasks/{task_id}")
def update_task(task_id: int, task: TaskUpdate, db: Session = Depends(get_db)):
    db_task = db.query(Task).filter(Task.id == task_id).first()
    if not db_task:
        raise HTTPException(status_code=404, detail="Task not found")

    update_data = task.dict(exclude_unset=True)

    # 変更履歴を記録
    for field, new_value in update_data.items():
        old_value = getattr(db_task, field)

        # 値が変更された場合のみ記録
        if old_value != new_value:
            activity = TaskActivity(
                task_id=task_id,
                field_name=field,
                old_value=str(old_value) if old_value is not None else None,
                new_value=str(new_value) if new_value is not None else None
            )
            db.add(activity)

        # フィールドを更新
        setattr(db_task, field, new_value)

    db.commit()
    db.refresh(db_task)
    return db_task
```

### 履歴取得 API

```python
@app.get("/tasks/{task_id}/activities")
def get_task_activities(task_id: int, db: Session = Depends(get_db)):
    # タスクの存在確認
    task = db.query(Task).filter(Task.id == task_id).first()
    if not task:
        raise HTTPException(status_code=404, detail="Task not found")

    # 履歴を新しい順に取得
    activities = (
        db.query(TaskActivity)
        .filter(TaskActivity.task_id == task_id)
        .order_by(TaskActivity.changed_at.desc())
        .all()
    )

    return activities
```

### Enum の記録

Enum 型のフィールドは **文字列に変換** して記録:

```python
# ❌ Bad: Enum オブジェクトをそのまま記録
old_value = db_task.status  # TaskStatus.TODO
new_value = str(old_value)   # "<TaskStatus.TODO>"（不正確）

# ✅ Good: .value で文字列を取得
old_value = db_task.status.value if db_task.status else None  # "TODO"
new_value = new_value.value if isinstance(new_value, Enum) else str(new_value)
```

def test_task_activity_logged_on_update(client):
    # タスク作成
    project = create_test_project(client)

    assert len(activities) == 1
    assert activities[0]["field_name"] == "status"

## Review（自己採点チェックリスト）

### 必須項目

- [ ] TaskActivity モデルが定義されている
- [ ] Alembic マイグレーションが作成・適用されている
- [ ] PUT /tasks/{id} で変更履歴が記録される
- [ ] 変更されたフィールドのみ記録される（変更なしの場合は記録しない）
- [ ] old_value と new_value が正しく記録される
- [ ] Enum が正しく文字列化されている（.value）
- [ ] GET /tasks/{id}/activities が実装されている
- [ ] 履歴が changed_at の降順でソートされている
- [ ] 存在しないタスク ID で 404 を返す
- [ ] 全テストが通る

### 加点項目

- [ ] changed_by フィールドがある（認証実装時に使用）
- [ ] 変更履歴が cascade delete される（タスク削除時）
- [ ] 複数フィールド更新時に全て記録されることをテストしている
- [ ] 変更なし（同じ値）の場合に activity が記録されないことをテストしている

### よくある間違い

- 全ての更新で activity を記録する（値が変更されていない場合も記録してしまう）
- Enum を str() で変換する（`"<TaskStatus.TODO>"` のような不正確な文字列になる）
- changed_at のソート順を指定しない（履歴が古い順になる）
- タスクの存在確認をしない（存在しないタスクの履歴で 500 エラー）
- relationship を定義しない（手動で JOIN を書く必要が出る）
- old_value / new_value を Text 型にしない（長い description が入らない）
