## Learn（概念理解）

### SQL 集計の基本

**COUNT, SUM, AVG, GROUP BY** を使ってデータを集計する:

```sql
-- タスクの総数
SELECT COUNT(*) FROM tasks WHERE project_id = 1;

-- ステータス別のタスク数
SELECT status, COUNT(*)
FROM tasks
WHERE project_id = 1
GROUP BY status;
```

### SQLAlchemy の func

SQLAlchemy で集計関数を使う:

```python
from sqlalchemy import func

# タスクの総数
total = db.query(func.count(Task.id)).filter(Task.project_id == project_id).scalar()

# ステータス別のタスク数
results = (
    db.query(Task.status, func.count(Task.id))
    .filter(Task.project_id == project_id)
    .group_by(Task.status)
    .all()
)

# results の形式:
# [("TODO", 3), ("IN_PROGRESS", 2), ("DONE", 5)]
```

### GROUP BY と集計結果の整形

```python
# ❌ Bad: タプルのリストをそのまま返す
[
  ("TODO", 3),
  ("IN_PROGRESS", 2),
  ("DONE", 5)
]

# ✅ Good: 辞書形式で返す
{
  "TODO": 3,
  "IN_PROGRESS": 2,
  "DONE": 5
}

# 実装
status_counts = {}
for status, count in results:
    status_counts[status.value] = count  # Enum の場合は .value

# または dict comprehension
status_counts = {status.value: count for status, count in results}
```

### サマリーAPI の設計

プロジェクトのサマリー情報を1つのエンドポイントで返す:

```python
GET /projects/{id}/summary

# レスポンス例
{
  "project_id": 1,
  "project_name": "Project A",
  "total_tasks": 10,
  "tasks_by_status": {
    "TODO": 3,
    "IN_PROGRESS": 2,
    "DONE": 5
  },
  "tasks_by_priority": {
    "LOW": 2,
    "MEDIUM": 5,
    "HIGH": 3
  }
}
```

### scalar() vs first() vs all()

| メソッド | 返り値 | 用途 |
|---------|--------|------|
| `scalar()` | 単一の値 | COUNT(*), SUM() など1つの値を取得 |
| `first()` | 1行（タプルまたはオブジェクト） | 1レコードを取得 |
| `all()` | 複数行のリスト | 複数レコードを取得 |

```python
# scalar: 1つの値
count = db.query(func.count(Task.id)).scalar()  # → 10

# first: 1行
task = db.query(Task).first()  # → <Task object>

# all: 複数行
tasks = db.query(Task).all()  # → [<Task>, <Task>, ...]

# GROUP BY の結果: all
results = db.query(Task.status, func.count(Task.id)).group_by(Task.status).all()
# → [("TODO", 3), ("DONE", 5)]
```

<details>
<summary>エンドポイントの実装例</summary>

```python
@app.get("/projects/{project_id}/summary", response_model=ProjectSummaryResponse)
def get_project_summary(project_id: int, db: Session = Depends(get_db)):
    # プロジェクトの存在確認
    project = db.query(Project).filter(Project.id == project_id).first()
    if not project:
        raise HTTPException(status_code=404, detail="Project not found")

    # タスク総数
    total_tasks = db.query(func.count(Task.id)).filter(Task.project_id == project_id).scalar()

    # ステータス別の集計
    status_results = (
        db.query(Task.status, func.count(Task.id))
        .filter(Task.project_id == project_id)
        .group_by(Task.status)
        .all()
    )
    tasks_by_status = {status.value: count for status, count in status_results}

    # 優先度別の集計
    priority_results = (
        db.query(Task.priority, func.count(Task.id))
        .filter(Task.project_id == project_id)
        .group_by(Task.priority)
        .all()
    )
    tasks_by_priority = {priority.value: count for priority, count in priority_results}

    return {
        "project_id": project.id,
        "project_name": project.name,
        "total_tasks": total_tasks or 0,
        "tasks_by_status": tasks_by_status,
        "tasks_by_priority": tasks_by_priority
    }
```

def test_project_summary_with_tasks(client):
    # プロジェクト作成
    project_response = client.post("/projects", json={"name": "Project A"})
    project_id = project_response.json()["id"]

    # 複数タスク作成
    client.post("/tasks", json={"project_id": project_id, "title": "Task 1", "status": "TODO", "priority": "HIGH"})
    client.post("/tasks", json={"project_id": project_id, "title": "Task 2", "status": "TODO", "priority": "LOW"})
    client.post("/tasks", json={"project_id": project_id, "title": "Task 3", "status": "DONE", "priority": "HIGH"})

    # サマリー取得
    response = client.get(f"/projects/{project_id}/summary")
    assert response.status_code == 200

    data = response.json()
    assert data["project_id"] == project_id
    assert data["project_name"] == "Project A"
    assert data["total_tasks"] == 3
    assert data["tasks_by_status"]["TODO"] == 2
    assert data["tasks_by_status"]["DONE"] == 1
    assert data["tasks_by_priority"]["HIGH"] == 2

## Review（自己採点チェックリスト）

### 必須項目

- [ ] GET /projects/{id}/summary エンドポイントが実装されている
- [ ] プロジェクトが存在しない場合、404 を返す
- [ ] total_tasks が正しく集計されている
- [ ] tasks_by_status がステータス別に集計されている（辞書形式）
- [ ] tasks_by_priority が優先度別に集計されている（辞書形式）
- [ ] タスク0件のプロジェクトで正しく動作する
- [ ] テストが全て通る

### 加点項目

- [ ] tasks_by_status / tasks_by_priority が全ての Enum 値を含む（0 で初期化）
- [ ] レスポンススキーマで型が明確に定義されている
- [ ] GROUP BY のクエリが効率的（N+1 クエリがない）
- [ ] テストでエッジケース（全タスクが同じステータス）を確認している

### よくある間違い

- `scalar()` を忘れて `first()` を使う（タプルが返ってくる）
- Enum を文字列に変換しない（`status` ではなく `status.value` を使う）
- タスク0件のとき、total_tasks が None になる（`or 0` でデフォルト値を設定）
- プロジェクトの存在確認をしない（存在しない ID で 500 エラー）
- GROUP BY の結果を辞書に変換しない（タプルのリストのまま返す）
- パフォーマンスを考慮せず、ループで複数クエリを発行する
