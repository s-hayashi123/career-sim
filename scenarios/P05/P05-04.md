## Learn（概念理解）

### Python集計 vs SQL集計

```python
# ❌ Bad: Python で集計（全データを取得してループ）
tasks = db.query(Task).filter(Task.project_id == project_id).all()

total = len(tasks)
todo_count = len([t for t in tasks if t.status == "TODO"])
in_progress_count = len([t for t in tasks if t.status == "IN_PROGRESS"])
done_count = len([t for t in tasks if t.status == "DONE"])

# 問題: 10,000タスク全てをメモリに読み込む

# ✅ Good: SQL で集計（DB側で処理）
from sqlalchemy import func

total = db.query(func.count(Task.id)).filter(Task.project_id == project_id).scalar()

status_counts = (
    db.query(Task.status, func.count(Task.id))
    .filter(Task.project_id == project_id)
    .group_by(Task.status)
    .all()
)

# メモリ使用量: 最小（集計結果のみ）
```

### GROUP BY の活用

```sql
-- Python集計の場合: 全行を取得
SELECT * FROM tasks WHERE project_id = 1;  -- 10,000行

-- SQL集計の場合: 集計結果のみ
SELECT status, COUNT(*) FROM tasks WHERE project_id = 1 GROUP BY status;
-- 結果: 3行（TODO: 3000, IN_PROGRESS: 2000, DONE: 5000）
```

### パフォーマンス比較

| 方式 | データ転送量 | メモリ使用量 | 処理速度 |
|------|------------|------------|---------|
| Python集計 | 10,000行 | 数MB | 遅い（1.5s） |
| SQL集計 | 3行 | 数KB | 速い（0.05s） |

改善率: **30倍**

### 複数の集計を同時実行

```python
# 複数の GROUP BY を一度に実行
from sqlalchemy import func, case

# ステータス別と優先度別を同時集計
result = (
    db.query(
        func.count(Task.id).label('total'),
        func.count(case((Task.status == TaskStatus.TODO, 1))).label('todo'),
        func.count(case((Task.status == TaskStatus.IN_PROGRESS, 1))).label('in_progress'),
        func.count(case((Task.status == TaskStatus.DONE, 1))).label('done'),
        func.count(case((Task.priority == TaskPriority.HIGH, 1))).label('high'),
        func.count(case((Task.priority == TaskPriority.MEDIUM, 1))).label('medium'),
        func.count(case((Task.priority == TaskPriority.LOW, 1))).label('low')
    )
    .filter(Task.project_id == project_id)
    .first()
)
```

## Practice（練習課題）

### 課題: Python集計をSQL集計に変更せよ

```python
# Before: Python集計
def get_order_stats(db: Session):
    orders = db.query(Order).all()

    total_orders = len(orders)
    total_revenue = sum(order.total for order in orders)
    avg_order_value = total_revenue / total_orders if total_orders > 0 else 0

    return {
        "total_orders": total_orders,
        "total_revenue": total_revenue,
        "avg_order_value": avg_order_value
    }
```

**Q1.** SQL集計に変更せよ（func.count, func.sum, func.avg）。
**Q2.** パフォーマンスはどのくらい改善するか？（10,000注文の場合）

### 模範解答

<details>
<summary>クリックで表示</summary>

**A1. SQL集計**:

```python
from sqlalchemy import func

def get_order_stats(db: Session):
    result = db.query(
        func.count(Order.id).label('total_orders'),
        func.sum(Order.total).label('total_revenue'),
        func.avg(Order.total).label('avg_order_value')
    ).first()

    return {
        "total_orders": result.total_orders or 0,
        "total_revenue": float(result.total_revenue or 0),
        "avg_order_value": float(result.avg_order_value or 0)
    }
```

**A2. パフォーマンス改善**:
- Before: 全10,000行を取得 → 数MB、1-2秒
- After: 集計結果のみ（1行）→ 数バイト、0.01秒
- 改善率: **100-200倍**

```python
# services/project_service.py
from sqlalchemy import func

class ProjectService:
    def get_project_summary(self, project_id: int) -> dict:
        # プロジェクトの存在確認
        project = self.get_project_by_id(project_id)

        # タスク総数
        total_tasks = (
            self.db.query(func.count(Task.id))
            .filter(Task.project_id == project_id)
            .scalar()
        ) or 0

        # ステータス別集計（SQL GROUP BY）
        status_results = (
            self.db.query(Task.status, func.count(Task.id))
            .filter(Task.project_id == project_id)
            .group_by(Task.status)
            .all()
        )
        tasks_by_status = {status.value: count for status, count in status_results}

        # 優先度別集計（SQL GROUP BY）
        priority_results = (
            self.db.query(Task.priority, func.count(Task.id))
            .filter(Task.project_id == project_id)
            .group_by(Task.priority)
            .all()
        )
        tasks_by_priority = {priority.value: count for priority, count in priority_results}

        return {
            "project_id": project.id,
            "project_name": project.name,
            "total_tasks": total_tasks,
            "tasks_by_status": tasks_by_status,
            "tasks_by_priority": tasks_by_priority
        }
```
