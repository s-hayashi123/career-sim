## Learn（概念理解）

### データ保持ポリシーとは

**古いデータをどう扱うか**の方針:

```
問題: ログデータが無限に増える
- task_activities: 1日1000件 → 1年で365,000件
- パフォーマンス低下
- ストレージ圧迫

解決策:
1. 定期削除（90日以上古いデータを削除）
2. アーカイブ（別テーブルに移動）
3. パーティショニング（日付ごとにテーブル分割）
```

### 定期削除スクリプト

```python
# scripts/cleanup_old_activities.py
from datetime import datetime, timedelta
from models import TaskActivity
from database import SessionLocal

def cleanup_old_activities(days=90):
    """90日以上古い活動ログを削除"""
    db = SessionLocal()

    cutoff_date = datetime.now() - timedelta(days=days)

    deleted_count = (
        db.query(TaskActivity)
        .filter(TaskActivity.changed_at < cutoff_date)
        .delete(synchronize_session=False)
    )

    db.commit()
    print(f"✅ Deleted {deleted_count} old activities (older than {days} days)")
    db.close()

if __name__ == "__main__":
    cleanup_old_activities(90)
```

### cron ジョブでの自動実行

```bash
# crontab -e
# 毎日 2:00 AM に実行
0 2 * * * cd /path/to/project && python scripts/cleanup_old_activities.py
```

### パーティショニング

**大きなテーブルを分割**:

```sql
-- 月ごとにパーティション
CREATE TABLE task_activities (
    id SERIAL PRIMARY KEY,
    task_id INTEGER,
    changed_at TIMESTAMP
) PARTITION BY RANGE (changed_at);

CREATE TABLE task_activities_2024_01 PARTITION OF task_activities
    FOR VALUES FROM ('2024-01-01') TO ('2024-02-01');

CREATE TABLE task_activities_2024_02 PARTITION OF task_activities
    FOR VALUES FROM ('2024-02-01') TO ('2024-03-01');

-- 古いパーティションを削除（高速）
DROP TABLE task_activities_2023_12;
```

### ADR-010 での方針記録

```markdown
# ADR-010: データ保持ポリシー

## Context
task_activities が無限に増え続ける。
現在: 100,000件、増加率: 1,000件/日

## Decision
90日保持ポリシー:
- 90日以上古い活動ログを定期削除
- 毎日 2:00 AM に自動実行（cron）

## Alternatives
1. アーカイブ: 複雑、ストレージ削減効果なし
2. パーティショニング: PostgreSQL 11+ 必要、今回は過剰

## Consequences
**Pros**:
- ストレージ削減
- クエリ性能維持
- シンプルな実装

**Cons**:
- 90日以上前のデータにアクセス不可
- 削除は不可逆
```

### ヒント

<details>
<summary>削除スクリプトの実装例</summary>

```python
# scripts/cleanup_old_activities.py
import sys
sys.path.append(".")

from datetime import datetime, timedelta
from models import TaskActivity
from database import SessionLocal

def cleanup_old_activities(days=90, dry_run=False):
    """90日以上古い活動ログを削除

    Args:
        days: 保持日数（デフォルト: 90）
        dry_run: True の場合、削除せずに件数だけ表示
    """
    db = SessionLocal()

    cutoff_date = datetime.now() - timedelta(days=days)

    print(f"Cleaning up activities older than {cutoff_date.date()} ({days} days ago)...")

    # 削除対象をカウント
    count_query = db.query(TaskActivity).filter(TaskActivity.changed_at < cutoff_date)
    count = count_query.count()

    print(f"Found {count} activities to delete")

    if dry_run:
        print("DRY RUN: No deletion performed")
    else:
        # 削除実行
        deleted_count = count_query.delete(synchronize_session=False)
        db.commit()
        print(f"✅ Deleted {deleted_count} old activities")

    db.close()

if __name__ == "__main__":
    import argparse

    parser = argparse.ArgumentParser()
    parser.add_argument("--days", type=int, default=90, help="Retention period in days")
    parser.add_argument("--dry-run", action="store_true", help="Dry run (no deletion)")
    args = parser.parse_args()

    cleanup_old_activities(days=args.days, dry_run=args.dry_run)
```
