# P01-01: ローカル環境セットアップ & README改善

> **シナリオ**: TLから「ローカル環境セットアップしてREADMEの不備を直して」
> **想定時間**: 3h
> **スキルタグ**: P8（ドキュメント・プロセス改善）

---

## Learn（概念理解）

### なぜ README が重要か

実務では、新しくチームに参加した人が **最初に読むのが README** 。ここが不完全だと:
- 環境構築に半日〜1日ムダにする
- 「動かないんですが...」と先輩に何度も聞く羽目になる
- チーム全員の生産性が下がる

### 良い README に含めるべき内容

1. **プロジェクト概要** — 何をするAPIか、1-2行で
2. **前提条件** — 必要なツールとバージョン（Python 3.x, PostgreSQL, etc.）
3. **環境構築手順** — コピペで動くコマンド列
4. **環境変数** — 必要な変数と説明
5. **起動方法** — ローカルでサーバーを立てるコマンド
6. **テスト実行方法** — テストの実行コマンド
7. **API ドキュメント** — Swagger UI の URL 等

### 実務での典型的な流れ

```
1. リポジトリを clone する
2. README を読む
3. 必要なツールをインストールする
4. 環境変数を設定する
5. DB をセットアップする
6. サーバーを起動する
7. テストを実行して通ることを確認する
```

この流れが README を読むだけで **完了できる** のが理想。

### Runbook とは

Runbook は「手順書」。README よりも詳細な、ステップバイステップのガイド。
実務では `docs/runbooks/` に置くことが多い。

例: `local-setup.md`（ローカル環境構築手順書）

```markdown
# ローカル環境構築

## 前提条件
- Python 3.11+
- PostgreSQL 15+

## 手順

### 1. リポジトリのクローン
git clone ...

### 2. 仮想環境の作成
python -m venv venv
source venv/bin/activate

### 3. 依存パッケージのインストール
pip install -r requirements.txt

（以下略）
```

**ポイント**: コマンドは全てコピペ可能にする。「適宜設定してください」は NG。

## Practice（練習課題）

### 課題: 以下の「ダメな README」を改善せよ

以下の内容の README があるとする:

```markdown
# My App

アプリです。

## セットアップ

Python入れて動かしてください。
```

**改善して、以下を含める:**

1. プロジェクト概要（1-2行）
2. 前提条件（ツール名 + バージョン）
3. セットアップ手順（コピペ可能なコマンド）
4. 起動方法
5. テスト実行方法

### 模範解答

<details>
<summary>クリックで表示</summary>

```markdown
# My App

ユーザー管理APIサーバー。FastAPI + PostgreSQL で構築。

## 前提条件

- Python 3.11+
- PostgreSQL 15+

## セットアップ

### 1. リポジトリのクローン

git clone https://github.com/example/my-app.git
cd my-app

### 2. 仮想環境の作成・有効化

python -m venv venv
source venv/bin/activate  # macOS/Linux

### 3. 依存パッケージのインストール

pip install -r requirements.txt

### 4. データベースの作成

createdb myapp

### 5. 環境変数の設定

export DATABASE_URL=postgresql://postgres:postgres@localhost:5432/myapp

### 6. マイグレーションの実行

alembic upgrade head

## 起動

uvicorn main:app --reload

APIドキュメント: http://localhost:8000/docs

## テスト

pytest tests/ -v
```

</details>

## Mission（本番課題）

### 状況

あなたは新メンバーとして `project-atlas-api` チームに参加しました。
TL から Slack で以下のメッセージが来ました:

> 「お疲れさま！まずローカル環境セットアップしてみて。README あるけど多分情報足りてないから、足りない部分は直しておいてくれると助かる。あと、セットアップ手順を Runbook として `docs/runbooks/local-setup.md` にまとめてほしい。」

### やること

1. **`project-atlas-api` をクローン（またはローカルで開く）**
2. **現状の README を読んで、何が足りないか確認する**
3. **実際にローカル環境をセットアップする**
   - Python 仮想環境を作る
   - `pip install -r requirements.txt` する
   - PostgreSQL に `atlas` データベースを作る
   - `alembic upgrade head` でマイグレーション実行
   - `uvicorn main:app --reload` でサーバー起動
   - `http://localhost:8000/docs` でSwagger UI確認
   - `pytest tests/ -v` でテスト実行
4. **GitHub Issue を作成する**
   - タイトル: `docs: README のセットアップ手順を改善する`
   - 本文: 現状の問題点と改善内容を記載

   <details>
   <summary>Issue の書き方例</summary>

   ```markdown
   ## 概要
   README のセットアップ手順が不足しており、新メンバーが環境構築でつまずく可能性がある。

   ## 背景
   現状の README には以下の情報が不足している:
   - 前提条件（Python, PostgreSQL のバージョン）
   - セットアップの詳細手順
   - データベースの作成方法
   - 環境変数の設定

   ## タスク
   - [ ] README に前提条件セクションを追加
   - [ ] ステップバイステップのセットアップ手順を追加
   - [ ] 起動方法・テスト実行方法を明記
   - [ ] Runbook (`docs/runbooks/local-setup.md`) を作成

   ## 受入条件
   - [ ] README を読むだけで環境構築ができる
   - [ ] 全コマンドがコピペ可能
   - [ ] Runbook に詳細手順が記載されている

   ## 参考資料
   - 既存の README.md
   ```

   **ポイント**:
   - 現状の問題点を具体的に記載
   - セットアップ手順の重要性を強調
   - チェックボックスで明確化
   </details>

5. **ブランチを作って作業する**
   - ブランチ名: `docs/proj01-readme-improvement`
6. **README.md を改善する**
   - 前提条件、セットアップ手順、起動方法、テスト実行方法を追加
7. **Runbook を作成する**
   - `docs/runbooks/local-setup.md`
   - ステップバイステップの詳細手順
8. **PR を作成する**
   - Issue を参照する
   - 変更内容を説明する

   <details>
   <summary>PR の書き方例</summary>

   ```markdown
   ## 概要
   README のセットアップ手順を改善し、新メンバーが迷わず環境構築できるようにしました。

   Closes #XX

   ## 変更内容
   - README に前提条件セクションを追加
   - ステップバイステップのセットアップ手順を追加
   - 起動方法・テスト実行方法を明記
   - `docs/runbooks/local-setup.md` を作成

   ## 動作確認
   - [ ] README の手順通りに環境構築ができる
   - [ ] 全コマンドがコピペで実行可能
   - [ ] サーバーが正常に起動する
   - [ ] テストが全て通る

   ## レビュー観点
   - [ ] セットアップ手順に抜け漏れがないか
   - [ ] コマンドが正確か
   - [ ] Runbook の内容が適切か
   ```

   **ポイント**:
   - Closes #XX で Issue を自動クローズ
   - 変更内容を箇条書き
   - 動作確認の方法を明示
   </details>

### Git 操作の参考

```bash
# ブランチ作成
git checkout -b docs/proj01-readme-improvement

# 変更をステージング
git add README.md docs/runbooks/local-setup.md

# コミット
git commit -m "docs: improve README setup instructions

- Add prerequisites section
- Add step-by-step setup instructions
- Add local-setup runbook

Closes #1"

# プッシュ
git push -u origin docs/proj01-readme-improvement

# PR 作成（GitHub CLI）
gh pr create --title "docs: improve README setup instructions" --body "..."
```

## Review（自己採点チェックリスト）

完了したら、以下をチェックしてください:

### 必須項目

- [x] README に前提条件（Python, PostgreSQL のバージョン）が記載されている
- [x] README のセットアップ手順がコピペで実行可能
- [x] README に起動方法が記載されている
- [x] README にテスト実行方法が記載されている
- [x] Runbook `docs/runbooks/local-setup.md` が存在する
- [x] Runbook に全手順がステップバイステップで記載されている
- [x] GitHub Issue が作成されている
- [x] PR が作成され、Issue を参照している
- [x] コミットメッセージが Conventional Commits 形式 (`docs:` prefix)

### 加点項目

- [ ] トラブルシューティングセクションがある（よくあるエラーと解決策）
- [ ] 環境変数の一覧と説明がある
- [x] API ドキュメント（Swagger UI）へのリンクがある
- [ ] PR の説明に Background / Changes / Testing セクションがある

### よくある間違い

- 「Python をインストールしてください」だけで具体的なバージョンを書かない
- `createdb` コマンドを書き忘れる（DB作成手順が抜ける）
- 環境変数 `DATABASE_URL` の設定手順が抜ける
- Runbook と README の内容が矛盾する
