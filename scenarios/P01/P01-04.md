## Learn（概念理解）

### .env.example の役割

アプリケーションの設定（DB接続先、APIキーなど）は **環境変数** で管理するのが基本。
`.env` ファイルに書くが、このファイルは **Git にコミットしてはいけない** （秘密情報が含まれるため）。

代わりに `.env.example` をコミットする:

```
# .env.example（コミットする。値はダミー）
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/atlas
APP_ENV=development

# .env（コミットしない。実際の値を入れる）
DATABASE_URL=postgresql://postgres:realpassword@localhost:5432/atlas
APP_ENV=development
```

**なぜ必要か:**
- 新メンバーが「どんな環境変数が必要か」一目でわかる
- `cp .env.example .env` してから値を変えるだけでセットアップ完了

### .gitignore の重要性

`.gitignore` は Git で **追跡しないファイル** を指定する。Python プロジェクトで最低限必要なもの:

```gitignore
# Python
__pycache__/
*.pyc
*.pyo
venv/
.venv/

# 環境変数
.env

# テスト
htmlcov/
.pytest_cache/
*.db

# IDE
.vscode/
.idea/
```

### Docker Compose とは

**複数のサービス（アプリ + DB）をまとめて起動する** ためのツール。

```yaml
# docker-compose.yml
services:
  db:
    image: postgres:15
    environment:
      POSTGRES_DB: atlas
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
    ports:
      - "5432:5432"

  api:
    build: .
    ports:
      - "8000:8000"
    depends_on:
      - db
    environment:
      DATABASE_URL: postgresql://postgres:postgres@db:5432/atlas
```

**なぜ必要か:**
- `docker compose up` だけで DB + API が起動する
- 「PostgreSQL のインストール方法がわからない」問題が消える
- チーム全員が **同じ環境** で開発できる

### Docker Compose がないとどうなるか

```
新メンバー: 「PostgreSQL ってどうやって入れるんですか？」
先輩A: 「brew install postgresql でいけるよ」
先輩B: 「自分は Docker で PostgreSQL だけ動かしてるけど」
先輩C: 「Windows なんですけど...」
```

→ Docker Compose があれば全員 `docker compose up db` で統一。

## Practice（練習課題）

### 課題1: .env.example を作成せよ

以下のコードから、必要な環境変数を読み取り `.env.example` を作成せよ:

```python
import os

DATABASE_URL = os.getenv("DATABASE_URL")
SECRET_KEY = os.getenv("SECRET_KEY", "dev-secret")
APP_ENV = os.getenv("APP_ENV", "development")
LOG_LEVEL = os.getenv("LOG_LEVEL", "INFO")
```

### 模範解答

<details>
<summary>クリックで表示</summary>

```env
# Database
DATABASE_URL=postgresql://postgres:postgres@localhost:5432/myapp

# Application
SECRET_KEY=dev-secret-change-in-production
APP_ENV=development
LOG_LEVEL=INFO
```

ポイント:
- デフォルト値があるものはそのまま記載
- `SECRET_KEY` のようなものは「本番では変えてね」とわかるダミー値
- `DATABASE_URL` はローカル開発用の値

</details>

### 課題2: docker-compose.yml を読んで質問に答えよ

```yaml
services:
  db:
    image: postgres:15
    environment:
      POSTGRES_DB: myapp
      POSTGRES_USER: user
      POSTGRES_PASSWORD: pass
    ports:
      - "5433:5432"
    volumes:
      - db-data:/var/lib/postgresql/data

volumes:
  db-data:
```

**Q1.** この PostgreSQL にホスト側から接続する場合、ポートは何番か？
**Q2.** `volumes` の `db-data` は何のためにあるか？
**Q3.** `docker compose down` するとデータはどうなるか？

### 模範解答

<details>
<summary>クリックで表示</summary>

**A1.** 5433（ホスト側のポート。コンテナ内では 5432）

**A2.** PostgreSQL のデータを永続化するため。`volumes` がないとコンテナを停止するたびにデータが消える。

**A3.** コンテナは停止・削除されるが、名前付きボリューム `db-data` は残るのでデータは消えない。`docker compose down -v` とすればボリュームも削除される。

</details>

## Mission（本番課題）

### 状況

P01-01 で環境構築をしているときに気付きました:

> 「このプロジェクト、`.env.example` がない。環境変数に何が必要かわからない。
> Docker Compose もないから、PostgreSQL を自分でインストール・設定しないといけない。
> 新しく入る人みんな同じ苦労をするから、ここを整備しよう。」

### やること

1. **GitHub Issue を作成する**
   - タイトル: `chore: .env.example と docker-compose.yml を追加する`

2. **ブランチを作成する**
   - `chore/proj01-env-docker-setup`

3. **`.env.example` を作成する**
   - `main.py` を読んで、必要な環境変数を確認する
   - ローカル開発用のデフォルト値を入れる

4. **`docker-compose.yml` を作成する**
   - PostgreSQL サービスを定義する
   - ポート、環境変数、ボリュームを設定する
   - データの永続化を設定する

5. **`.gitignore` を整備する**
   - Python プロジェクトに適した `.gitignore` を作成する
   - `.env` を必ず含める

6. **README を更新する**
   - Docker Compose を使ったセットアップ手順を追記する
   ```
   ## Docker を使ったセットアップ
   docker compose up db -d
   ```

7. **PR を作成する**

### ヒント

<details>
<summary>main.py のどこを見ればいいか</summary>

```python
DATABASE_URL = os.getenv("DATABASE_URL", "postgresql://postgres:postgres@localhost:5432/atlas")
```

この行から、`DATABASE_URL` が必要な環境変数だとわかります。

### 加点項目

### よくある間違い

- `.env` を Git にコミットしてしまう（秘密情報漏洩）
- `docker-compose.yml` でポートマッピングを忘れる（ホストからアクセスできない）
- PostgreSQL のデータベース名がコードと `docker-compose.yml` で異なる
- `.env.example` に本物のパスワードを書いてしまう
