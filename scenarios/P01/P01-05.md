## Learn（概念理解）

### OpenAPI（Swagger）とは

REST API の **仕様書** 。FastAPI は Pydantic スキーマから **自動生成** してくれる。

- `http://localhost:8000/docs` → Swagger UI（インタラクティブなAPI画面）
- `http://localhost:8000/openapi.json` → JSON形式の仕様

### なぜドキュメントを整備するのか

フロントエンド開発者がAPIを使うとき:

```
❌ ドキュメントなし:
  「この email って必須？フォーマットは？」→ バックエンドに聞く → 待ち時間発生

✅ ドキュメントあり:
  Swagger UI を見る → 「email: string, required, example: taro@example.com」→ すぐ実装
```

### Pydantic スキーマで説明を追加する方法

```python
# ❌ 説明なし（現状）
class UserCreate(BaseModel):
    name: str
    email: str

# ✅ 説明あり（改善後）
class UserCreate(BaseModel):
    name: str = Field(
        ...,
        description="ユーザーの表示名",
        example="Taro Yamada",
        min_length=1,
        max_length=100,
    )
    email: str = Field(
        ...,
        description="ユーザーのメールアドレス",
        example="taro@example.com",
    )
```

### Field() の主なパラメータ

| パラメータ | 説明 | 例 |
|-----------|------|-----|
| `description` | フィールドの説明 | `"ユーザーの表示名"` |
| `example` | サンプル値 | `"Taro Yamada"` |
| `min_length` | 最小文字数 | `1` |
| `max_length` | 最大文字数 | `100` |
| `ge` | 最小値（数値） | `1` |
| `le` | 最大値（数値） | `100` |

### エンドポイントにも説明を追加

```python
@app.post(
    "/users",
    response_model=UserResponse,
    summary="ユーザー作成",
    description="新しいユーザーを作成する。name と email は必須。",
    status_code=201,
    tags=["Users"],
)
def create_user(user: UserCreate, db: Session = Depends(get_db)):
    ...
```

### タグでエンドポイントを分類

```python
app = FastAPI(title="Project Atlas API", version="0.1.0")

# タグでグループ化
@app.get("/users", tags=["Users"])
@app.post("/users", tags=["Users"])
@app.get("/teams", tags=["Teams"])
```

Swagger UI でタグごとにまとまって表示される。

## Practice（練習課題）

### 課題: 以下のスキーマに `Field()` を追加せよ

```python
class ProductCreate(BaseModel):
    name: str
    price: int
    description: str
```

要件:
- `name`: 必須、1-200文字、例 "ノートPC"
- `price`: 必須、1以上、例 98000
- `description`: 任意、500文字以内、例 "高性能ノートパソコン"

### 模範解答

<details>
<summary>クリックで表示</summary>

```python
from pydantic import BaseModel, Field
from typing import Optional

class ProductCreate(BaseModel):
    name: str = Field(
        ...,
        description="商品名",
        example="ノートPC",
        min_length=1,
        max_length=200,
    )
    price: int = Field(
        ...,
        description="価格（円）",
        example=98000,
        ge=1,
    )
    description: Optional[str] = Field(
        None,
        description="商品の説明",
        example="高性能ノートパソコン",
        max_length=500,
    )
```

</details>

## Mission（本番課題）

### 状況

TL から Slack:

> 「フロントチームから Swagger UI 見たけど description も example もなくて何もわからんって言われた。整備してくれない？タグ分けもしてほしい。」

### やること

1. **GitHub Issue を作成する**
   - タイトル: `docs: OpenAPI ドキュメントを整備する`

2. **ブランチを作成する**
   - `docs/proj01-openapi-improvement`

3. **Swagger UI の現状を確認する**
   - `http://localhost:8000/docs` を開いて現状を確認

4. **Pydantic スキーマに `Field()` を追加する**
   - `UserCreate`: name, email に description, example を追加
   - `UserUpdate`: 同様
   - `UserPatch`: 同様
   - `TeamCreate`: name, description に description, example を追加
   - `TeamMemberCreate`: user_id, role に description, example を追加
   - レスポンスモデル（`UserResponse` 等）にも description を追加

5. **エンドポイントに説明を追加する**
   - 各エンドポイントに `summary`, `tags` を追加
   - Tags: `["Users"]`, `["Teams"]`, `["Team Members"]`, `["Health"]`

6. **FastAPI アプリにメタ情報を追加する**
   ```python
   app = FastAPI(
       title="Project Atlas API",
       description="チーム・ユーザー管理プラットフォーム API",
       version="0.1.0",
   )
   ```

7. **Swagger UI で確認する**
   - 全エンドポイントに説明がある
   - タグでグループ化されている
   - example 値で Try it out が試せる

8. **PR を作成する**

### 加点項目

- [ ] エンドポイントに `summary` が追加されている
- [ ] レスポンスモデルにも description が追加されている
- [ ] `response_model` のレスポンス例が Swagger UI に表示される
- [ ] Swagger UI のスクリーンショットを PR に添付している

### よくある間違い

- `Field()` の `...` （必須マーカー）を忘れてデフォルト値が `None` になる
- `example` の値が実際のバリデーションに通らない値にしてしまう
- description を日本語/英語で混在させる（統一する）
- タグ名に表記揺れ（`"users"` と `"Users"` が混在）
