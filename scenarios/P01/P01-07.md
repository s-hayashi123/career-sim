## Learn（概念理解）

### ヘルスチェックとは

`/health` エンドポイントは **「このサービスは正常に動いているか」** を返す。
ロードバランサーや監視ツールが定期的に叩いて、異常があればアラートを出す。

### なぜ DB 確認が必要か

```
# ❌ 今の実装
GET /health → {"status": "ok"}

# これだと:
# - API プロセスは動いている ✅
# - でも DB が落ちている ❌ → ユーザーリクエストは全て500
# - なのに health は "ok" と返す 😱
```

**DB に接続できないのに "ok" を返すのは嘘** 。

### 良いヘルスチェックの設計

```python
@app.get("/health")
def health_check(db: Session = Depends(get_db)):
    try:
        db.execute(text("SELECT 1"))
        return {
            "status": "healthy",
            "database": "connected",
        }
    except Exception as e:
        return JSONResponse(
            status_code=503,
            content={
                "status": "unhealthy",
                "database": "disconnected",
            },
        )
```

**ポイント:**
- DB に `SELECT 1` を実行して接続確認
- 異常時は `503 Service Unavailable` を返す
- ステータスコードで異常を伝える（ロードバランサーはステータスコードを見る）

### ヘルスチェックのレベル

| レベル | 確認内容 | 例 |
|-------|---------|-----|
| Liveness | プロセスが生きているか | `{"status": "ok"}` |
| Readiness | リクエストを受け付けられるか | DB接続、依存サービスの確認 |
| Startup | 起動完了したか | 初期化処理の完了確認 |

実務では最低でも **Readiness** レベル（DB確認込み）が必要。

## Practice（練習課題）

### 課題: 以下のヘルスチェックの問題点を指摘し、改善せよ

```python
@app.get("/health")
def health():
    return {"status": "ok"}
```

このサービスは PostgreSQL と Redis に依存しているとする。

### 模範解答

<details>
<summary>クリックで表示</summary>

**問題点:**
1. DB（PostgreSQL）の接続確認をしていない
2. Redis の接続確認をしていない
3. 異常時でも 200 OK を返してしまう

**改善:**
```python
from fastapi.responses import JSONResponse
from sqlalchemy import text

@app.get("/health")
def health(db: Session = Depends(get_db)):
    health_status = {"status": "healthy"}
    is_healthy = True

    # DB確認
    try:
        db.execute(text("SELECT 1"))
        health_status["database"] = "connected"
    except Exception:
        health_status["database"] = "disconnected"
        is_healthy = False

    # Redis確認
    try:
        redis_client.ping()
        health_status["redis"] = "connected"
    except Exception:
        health_status["redis"] = "disconnected"
        is_healthy = False

    if not is_healthy:
        health_status["status"] = "unhealthy"
        return JSONResponse(status_code=503, content=health_status)

    return health_status
```

</details>

   - 正常時のテスト（DB接続成功）

### ヒント

`text()` でラップしないと SQLAlchemy 2.x ではエラーになります。

## Review（自己採点チェックリスト）

### 必須項目

- [ ] `/health` が DB 接続を確認している
- [ ] DB 接続成功時: `200` + `{"status": "healthy", "database": "connected"}`
- [ ] DB 接続失敗時: `503` + `{"status": "unhealthy", ...}`
- [ ] テストが追加されている
- [ ] GitHub Issue と PR が作成されている

### 加点項目

- [ ] `text("SELECT 1")` を使用している（SQLAlchemy 2.x 対応）
- [ ] DB 接続失敗時のテストがある
- [ ] レスポンスに DB 以外の情報（バージョンなど）も含めている
- [ ] 例外の詳細はログに出力し、レスポンスには含めない（情報漏洩防止）

### よくある間違い

- 例外発生時に `raise` してしまい 500 になる（503 を返すべき）
- 例外のエラーメッセージをそのままレスポンスに含める（セキュリティリスク）
- `SELECT 1` ではなく大きなクエリを実行してしまう（ヘルスチェックは軽量に）
- テストを書かない
