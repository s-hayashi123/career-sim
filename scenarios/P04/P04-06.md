## Learn（概念理解）

### ミックスイン（Mixin）とは

**複数のクラスで共通する機能を再利用する**パターン:

```python
# ❌ Bad: 全モデルで id, created_at を繰り返し定義
class User(Base):
    __tablename__ = "users"
    id = Column(Integer, primary_key=True)
    created_at = Column(DateTime, server_default=func.now())
    name = Column(String(255))

class Project(Base):
    __tablename__ = "projects"
    id = Column(Integer, primary_key=True)
    created_at = Column(DateTime, server_default=func.now())
    name = Column(String(255))

class Task(Base):
    __tablename__ = "tasks"
    id = Column(Integer, primary_key=True)
    created_at = Column(DateTime, server_default=func.now())
    title = Column(String(255))

# ✅ Good: ミックスインで共通化
class TimestampMixin:
    """タイムスタンプフィールドのミックスイン"""
    id = Column(Integer, primary_key=True, index=True)
    created_at = Column(DateTime, nullable=False, server_default=func.now())
    updated_at = Column(DateTime, nullable=False, server_default=func.now(), onupdate=func.now())

class User(Base, TimestampMixin):
    __tablename__ = "users"
    name = Column(String(255))

class Project(Base, TimestampMixin):
    __tablename__ = "projects"
    name = Column(String(255))

class Task(Base, TimestampMixin):
    __tablename__ = "tasks"
    title = Column(String(255))
```

### SQLAlchemy のミックスインパターン

```python
from sqlalchemy import Column, Integer, DateTime, String
from sqlalchemy.sql import func
from sqlalchemy.ext.declarative import declarative_base

Base = declarative_base()

class TimestampMixin:
    """全モデル共通のタイムスタンプ"""
    created_at = Column(DateTime, nullable=False, server_default=func.now())
    updated_at = Column(DateTime, nullable=False, server_default=func.now(), onupdate=func.now())

class PrimaryKeyMixin:
    """全モデル共通の主キー"""
    id = Column(Integer, primary_key=True, index=True)

# 複数ミックスインの組み合わせ
class BaseModel(Base, PrimaryKeyMixin, TimestampMixin):
    """全モデルの基底クラス"""
    __abstract__ = True  # このクラス自体はテーブルを作らない

class User(BaseModel):
    __tablename__ = "users"
    name = Column(String(255), nullable=False)
    # id, created_at, updated_at は自動的に継承される
```

### updated_at の自動更新

```python
# onupdate で更新時に自動的にタイムスタンプを更新
updated_at = Column(DateTime, nullable=False, server_default=func.now(), onupdate=func.now())

# テスト
user = User(name="Alice")
db.add(user)
db.commit()
print(user.created_at)  # 2024-01-01 10:00:00

# 更新
user.name = "Alicia"
db.commit()
print(user.updated_at)  # 2024-01-01 10:05:00（自動更新）
```

### マイグレーションでのカラム追加

既存のモデルに updated_at を追加する場合:

```python
# alembic/versions/xxxx_add_updated_at.py
def upgrade():
    # User
    op.add_column('users', sa.Column('updated_at', sa.DateTime(), nullable=False, server_default=sa.func.now()))

    # Project
    op.add_column('projects', sa.Column('updated_at', sa.DateTime(), nullable=False, server_default=sa.func.now()))

    # Task
    op.add_column('tasks', sa.Column('updated_at', sa.DateTime(), nullable=False, server_default=sa.func.now()))

def downgrade():
    op.drop_column('users', 'updated_at')
    op.drop_column('projects', 'updated_at')
    op.drop_column('tasks', 'updated_at')
```

## Practice（練習課題）

### 課題: 以下のモデルを BaseModel ミックスインで共通化せよ

```python
class Product(Base):
    __tablename__ = "products"
    id = Column(Integer, primary_key=True)
    created_at = Column(DateTime, server_default=func.now())
    name = Column(String(255))
    price = Column(Float)

class Category(Base):
    __tablename__ = "categories"
    id = Column(Integer, primary_key=True)
    created_at = Column(DateTime, server_default=func.now())
    name = Column(String(255))

class Order(Base):
    __tablename__ = "orders"
    id = Column(Integer, primary_key=True)
    created_at = Column(DateTime, server_default=func.now())
    user_id = Column(Integer, ForeignKey("users.id"))
    total = Column(Float)
```

**Q1.** BaseModel ミックスインを作成せよ（id, created_at, updated_at）。
**Q2.** 各モデルをリファクタリングせよ。
**Q3.** マイグレーションで updated_at を追加するコードを書け。

## Mission（本番課題）

### 状況

コードを見直していると、User, Project, Task の全モデルで `id` と `created_at` が重複していることに気づきました。さらに、`updated_at` がないため、いつ更新されたかわかりません。

TL に相談すると:

> 「BaseModel ミックスインを作って、全モデルで共通化しよう。ついでに updated_at も追加して。」

### やること

1. **GitHub Issue を作成する**
   - タイトル: `refactor: BaseModel ミックスインと updated_at を追加`
   - 本文: 重複フィールド、updated_at の必要性

2. **ブランチを作成する**
   ```bash
   git checkout -b refactor/proj04-base-model-mixin
   ```

3. **BaseModel ミックスインを作成する**
   - `models.py` に追加:
     - `__abstract__ = True`（テーブルを作らない）
     - `id`: primary key, index
     - `created_at`: server_default=func.now()
     - `updated_at`: server_default=func.now(), onupdate=func.now()

4. **全モデルをリファクタリングする**
   - User, Project, Task が BaseModel を継承
   - 重複する id, created_at を削除

5. **Alembic マイグレーションを作成する**
   - updated_at カラムを全テーブルに追加
   - server_default=func.now() を設定

6. **マイグレーションを適用する**
   ```bash
   alembic revision --autogenerate -m "add updated_at to all tables"
   alembic upgrade head
   ```

7. **Pydantic スキーマを更新する**
   - UserResponse, ProjectResponse, TaskResponse に updated_at を追加

8. **テストを更新する**
   - updated_at が返されることを確認

9. **全テストが通ることを確認する**
   ```bash
   pytest tests/ -v
   ```

10. **PR を作成する**

### Git 操作の参考

```bash
git checkout -b refactor/proj04-base-model-mixin

git add models.py schemas.py alembic/versions/*.py tests/

git commit -m "refactor: introduce BaseModel mixin with updated_at
```

## Review（自己採点チェックリスト）

### 必須項目

- [ ] BaseModel ミックスインが作成されている
- [ ] BaseModel に id, created_at, updated_at がある
- [ ] `__abstract__ = True` が設定されている
- [ ] 全モデルが BaseModel を継承している
- [ ] 重複する id, created_at が削除されている
- [ ] updated_at カラムのマイグレーションが作成・適用されている
- [ ] Pydantic スキーマに updated_at が追加されている
- [ ] 全テストが通る

### 加点項目

- [ ] onupdate=func.now() が設定されている（自動更新）
- [ ] マイグレーションで server_default を設定している
- [ ] テストで updated_at の自動更新を確認している

### よくある間違い

- `__abstract__ = True` を忘れる（base_model テーブルが作られる）
- onupdate を設定しない（手動で updated_at を更新する必要がある）
- 既存データの updated_at が NULL になる（server_default がない）
- Pydantic スキーマを更新しない（レスポンスに updated_at が含まれない）
- マイグレーションを適用しない（DB にカラムがない）
