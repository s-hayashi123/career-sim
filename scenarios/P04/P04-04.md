## Learn（概念理解）

### DRY原則（Don't Repeat Yourself）

同じコードを繰り返さない:

```python
# ❌ Bad: 同じバリデーションを3箇所にコピペ
class UserCreate(BaseModel):
    name: str
    email: str

    @validator("email")
    def validate_email(cls, v):
        if not re.match(r"^[\w\.-]+@[\w\.-]+\.\w+$", v):
            raise ValueError("Invalid email format")
        return v

class ProjectCreate(BaseModel):
    name: str
    contact_email: str

    @validator("contact_email")
    def validate_email(cls, v):
        if not re.match(r"^[\w\.-]+@[\w\.-]+\.\w+$", v):
            raise ValueError("Invalid email format")
        return v

# ✅ Good: カスタム型を作成
from pydantic import validator, Field
import re

class EmailStr(str):
    """メールアドレス専用型"""
    @classmethod
    def __get_validators__(cls):
        yield cls.validate

    @classmethod
    def validate(cls, v):
        if not isinstance(v, str):
            raise TypeError("string required")
        if not re.match(r"^[\w\.-]+@[\w\.-]+\.\w+$", v):
            raise ValueError("Invalid email format")
        return cls(v)

# 再利用
class UserCreate(BaseModel):
    name: str
    email: EmailStr  # カスタム型を使用

class ProjectCreate(BaseModel):
    name: str
    contact_email: EmailStr  # 同じ型を再利用
```

### Pydantic のカスタムバリデータ

```python
# validators.py
from pydantic import validator

def validate_name_length(v: str) -> str:
    """名前の長さをバリデート（2-255文字）"""
    if len(v) < 2:
        raise ValueError("Name must be at least 2 characters")
    if len(v) > 255:
        raise ValueError("Name must not exceed 255 characters")
    return v

def validate_positive_integer(v: int) -> int:
    """正の整数をバリデート"""
    if v <= 0:
        raise ValueError("Must be a positive integer")
    return v

# 使用例
class UserCreate(BaseModel):
    name: str

    _validate_name = validator("name", allow_reuse=True)(validate_name_length)

class ProjectCreate(BaseModel):
    name: str

    _validate_name = validator("name", allow_reuse=True)(validate_name_length)
```

### Field の活用

```python
from pydantic import BaseModel, Field

# ❌ Bad: 制約が散在
class TaskCreate(BaseModel):
    title: str
    priority: int

    @validator("title")
    def validate_title(cls, v):
        if len(v) < 1 or len(v) > 255:
            raise ValueError("Title must be 1-255 characters")
        return v

    @validator("priority")
    def validate_priority(cls, v):
        if v < 1 or v > 5:
            raise ValueError("Priority must be 1-5")
        return v

# ✅ Good: Field で制約を定義
class TaskCreate(BaseModel):
    title: str = Field(..., min_length=1, max_length=255, description="タスクのタイトル")
    priority: int = Field(..., ge=1, le=5, description="優先度（1-5）")
```

### 共通バリデーション関数の配置

```python
# utils/validators.py
import re

def is_valid_email(email: str) -> bool:
    """メールアドレスの形式チェック"""
    pattern = r"^[\w\.-]+@[\w\.-]+\.\w+$"
    return re.match(pattern, email) is not None

def is_valid_url(url: str) -> bool:
    """URL の形式チェック"""
    pattern = r"^https?://[\w\.-]+\.\w+"
    return re.match(pattern, url) is not None

# schemas.py で使用
from utils.validators import is_valid_email

class UserCreate(BaseModel):
    email: str

    @validator("email")
    def validate_email(cls, v):
        if not is_valid_email(v):
            raise ValueError("Invalid email format")
        return v
```

## Mission（本番課題）

### 状況

サービス層のリファクタリングを進めていると、以下の重複に気づきました:

1. **メールアドレスのバリデーション**: User, Project の contact_email で同じ正規表現
2. **名前の長さチェック**: User.name, Project.name, Task.title で同じロジック（1-255文字）
3. **正の整数チェック**: Task.priority, 各種 ID で同じロジック

TL に相談すると:

> 「いいところに気づいたね！共通バリデータとカスタム型を作って、DRY にしよう。」

### やること

1. **GitHub Issue を作成する**
   - タイトル: `refactor: 共通バリデータとカスタムPydantic型を作成`
   - 本文: 重複しているバリデーションの箇所

2. **ブランチを作成する**
   ```bash
   git checkout -b refactor/proj04-common-validators
   ```

3. **共通バリデータを作成する**
   - ファイル: `utils/validators.py`
   - 関数:
     - `validate_email_format(v: str) -> str`
     - `validate_name_length(v: str) -> str`
     - `validate_positive_integer(v: int) -> int`

4. **カスタムPydantic型を作成する**
   - ファイル: `types/custom_types.py`
   - 型:
     - `EmailStr`: メールアドレス専用型
     - `NameStr`: 名前専用型（1-255文字）
     - `PositiveInt`: 正の整数専用型

5. **schemas.py をリファクタリングする**
   - カスタム型を使用
   - 重複バリデータを削除

6. **テストを追加する**
   - `tests/unit/test_validators.py`
   - `tests/unit/test_custom_types.py`
   - 各バリデータ・カスタム型の正常系・異常系

7. **既存のテストが通ることを確認する**
   ```bash
   pytest tests/ -v
   ```

8. **PR を作成する**

```python
# types/custom_types.py
import re
from pydantic import validator

class EmailStr(str):
    """メールアドレス専用型"""
    @classmethod
    def __get_validators__(cls):
        yield cls.validate

    @classmethod
    def validate(cls, v):
        if not isinstance(v, str):
            raise TypeError("string required")
        pattern = r"^[\w\.-]+@[\w\.-]+\.\w+$"
        if not re.match(pattern, v):
            raise ValueError("Invalid email format")
        return cls(v)

class NameStr(str):
    """名前専用型（1-255文字）"""
    @classmethod
    def __get_validators__(cls):
        yield cls.validate

    @classmethod
    def validate(cls, v):
        if not isinstance(v, str):
            raise TypeError("string required")
        if len(v) < 1 or len(v) > 255:
            raise ValueError("Name must be 1-255 characters")
        return cls(v)

class PositiveInt(int):
    """正の整数専用型"""
    @classmethod
    def __get_validators__(cls):
        yield cls.validate

    @classmethod
    def validate(cls, v):
        if not isinstance(v, int):
            raise TypeError("integer required")
        if v <= 0:
            raise ValueError("Must be a positive integer")
        return cls(v)
```

```python
import pytest
from types.custom_types import EmailStr, NameStr, PositiveInt

def test_email_str_valid():

    assert email == "user@example.com"

def test_email_str_invalid():
    with pytest.raises(ValueError, match="Invalid email"):

def test_name_str_valid():
    name = NameStr.validate("Alice")
    assert name == "Alice"

def test_name_str_too_short():
    with pytest.raises(ValueError, match="1-255 characters"):
        NameStr.validate("")

def test_positive_int_valid():
    num = PositiveInt.validate(42)
    assert num == 42

def test_positive_int_zero():
    with pytest.raises(ValueError, match="positive integer"):
```

## Review（自己採点チェックリスト）

### 必須項目

- [ ] `utils/validators.py` が作成されている
- [ ] `types/custom_types.py` が作成されている
- [ ] 重複バリデーションが削除されている
- [ ] スキーマがカスタム型を使用している
- [ ] カスタム型の単体テストがある
- [ ] 既存のテストが全て通る

### 加点項目

- [ ] カスタム型に docstring がある
- [ ] Field の description と組み合わせている
- [ ] エラーメッセージが具体的
- [ ] テストが網羅的（正常系・異常系・境界値）

### よくある間違い

- カスタム型を作らず、バリデータ関数だけにする（型安全性が低い）
- すべてをカスタム型にする（単純なものは Field で十分）
- テストを書かない（リファクタリングでバグを混入）
- 既存のバリデーションを削除し忘れる（重複が残る）
