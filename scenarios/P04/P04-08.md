## Learn（概念理解）

### 依存性注入（Dependency Injection）とは

**依存関係を外部から注入する**パターン。テスタビリティと柔軟性が向上:

```python
# ❌ Bad: 依存関係がハードコード
class UserService:
    def __init__(self):
        self.db = SessionLocal()  # ハードコード

    def create_user(self, user_data):
        ...

# テストが困難（実際の DB に接続してしまう）

# ✅ Good: 依存性注入
class UserService:
    def __init__(self, db: Session):  # 外部から注入
        self.db = db

# テストで DB をモックに差し替え可能
mock_db = MagicMock()
service = UserService(mock_db)
```

### FastAPI の Depends

FastAPI では **Depends** で依存性注入:

```python
from fastapi import Depends
from sqlalchemy.orm import Session

# DB セッションを生成する関数
def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

# ルートハンドラで注入
@app.get("/users/{user_id}")
def get_user(
    user_id: int,
    db: Session = Depends(get_db)  # 依存性注入
):
    user = db.query(User).filter(User.id == user_id).first()
    return user
```

**メリット**:
- テスト時に DB を差し替え可能
- セッションのライフサイクル管理が自動化
- 複数のエンドポイントで再利用

### サービス層の依存性注入

```python
# ❌ Bad: サービスインスタンスをルートハンドラで毎回生成
@app.post("/users")
def create_user(user: UserCreate, db: Session = Depends(get_db)):
    service = UserService(db)  # 毎回生成
    return service.create_user(user)

@app.get("/users")
def list_users(db: Session = Depends(get_db)):
    service = UserService(db)  # 毎回生成
    return service.list_users()

# ✅ Good: サービスも依存性注入
def get_user_service(db: Session = Depends(get_db)) -> UserService:
    return UserService(db)

@app.post("/users")
def create_user(
    user: UserCreate,
    service: UserService = Depends(get_user_service)  # サービスを注入
):
    return service.create_user(user)

@app.get("/users")
def list_users(service: UserService = Depends(get_user_service)):
    return service.list_users()
```

### テスタビリティの向上

```python
# tests/conftest.py
@pytest.fixture
def test_db():
    """テスト用 DB セッション"""
    db = TestSessionLocal()
    try:
        yield db
    finally:
        db.close()

def override_get_db(test_db):
    """get_db を上書き"""
    def _get_db():
        yield test_db
    return _get_db

# テストで使用
def test_create_user(test_db):
    app.dependency_overrides[get_db] = override_get_db(test_db)

    response = client.post("/users", json={"name": "Alice", "email": "alice@example.com"})
    assert response.status_code == 201
```

### Depends の階層化

```python
def get_db():
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

def get_user_service(db: Session = Depends(get_db)) -> UserService:
    return UserService(db)

def get_current_user(
    token: str = Depends(oauth2_scheme),
    service: UserService = Depends(get_user_service)
) -> User:
    """認証されたユーザーを取得"""
    user = service.verify_token(token)
    if not user:
        raise HTTPException(status_code=401, detail="Invalid token")
    return user

# ルートハンドラ
@app.get("/me")
def get_me(current_user: User = Depends(get_current_user)):
    return current_user
```

## Mission（本番課題）

### 状況

サービス層をリファクタリングしてきましたが、各ルートハンドラで毎回 `service = UserService(db)` と書いています。これを依存性注入パターンに変更したいと TL に提案すると:

> 「いいね！FastAPI の Depends を使って、サービス層も DI 化しよう。テストもやりやすくなるし。」

### やること

1. **GitHub Issue を作成する**
   - タイトル: `refactor: サービス層を依存性注入パターンに変更`

2. **ブランチを作成する**
   ```bash
   git checkout -b refactor/proj04-service-di
   ```

3. **サービス用の依存性関数を作成する**
   - `dependencies.py` を作成
   - 関数:
     - `get_db() -> Generator[Session, None, None]`（既存）
     - `get_user_service(db: Session = Depends(get_db)) -> UserService`
     - `get_project_service(db: Session = Depends(get_db)) -> ProjectService`
     - `get_task_service(db: Session = Depends(get_db)) -> TaskService`

4. **全ルートハンドラをリファクタリングする**
   - User エンドポイント: `service: UserService = Depends(get_user_service)`
   - Project エンドポイント: `service: ProjectService = Depends(get_project_service)`
   - Task エンドポイント: `service: TaskService = Depends(get_task_service)`

5. **テストを更新する**
   - `conftest.py` で `get_db` を override
   - 全テストが通ることを確認

6. **全テストが通ることを確認する**
   ```bash
   pytest tests/ -v
   ```

7. **PR を作成する**

<details>
<summary>dependencies.py の実装例</summary>

```python
# dependencies.py
from sqlalchemy.orm import Session
from typing import Generator
from database import SessionLocal
from services.user_service import UserService
from services.project_service import ProjectService
from services.task_service import TaskService

def get_db() -> Generator[Session, None, None]:
    """DB セッションを生成"""
    db = SessionLocal()
    try:
        yield db
    finally:
        db.close()

def get_user_service(db: Session = Depends(get_db)) -> UserService:
    """UserService を生成"""
    return UserService(db)

def get_project_service(db: Session = Depends(get_db)) -> ProjectService:
    """ProjectService を生成"""
    return ProjectService(db)

def get_task_service(db: Session = Depends(get_db)) -> TaskService:
    """TaskService を生成"""
    return TaskService(db)
```

## Review（自己採点チェックリスト）

### 必須項目

- [ ] dependencies.py が作成されている
- [ ] get_user_service, get_project_service, get_task_service がある
- [ ] 全ルートハンドラがサービスを Depends で注入している
- [ ] `service = XxxService(db)` が削除されている
- [ ] テストで get_db を override している
- [ ] 全テストが通る

### 加点項目

- [ ] 型ヒントが正しい（Generator[Session, None, None]）
- [ ] docstring がある
- [ ] テストでの override が fixture で共通化されている

### よくある間違い

- 一部のエンドポイントだけ DI にする（一貫性がない）
- dependencies.py を作らず、main.py に全て書く
- テストで dependency_overrides を clear しない（テスト間で影響）
- get_db の yield を忘れる（finally が実行されない）
