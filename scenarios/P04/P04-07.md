## Learn（概念理解）

### なぜ型ヒントが重要か

**静的型チェック**で実行前にバグを検出:

```python
# ❌ Bad: 型ヒントなし（実行時エラー）
def get_user_by_id(user_id):
    user = db.query(User).filter(User.id == user_id).first()
    return user

# 実行時にエラー
result = get_user_by_id("abc")  # 文字列を渡してしまう
# → TypeError: '>' not supported between instances of 'int' and 'str'

# ✅ Good: 型ヒントあり（mypy が検出）
def get_user_by_id(user_id: int) -> User | None:
    user = db.query(User).filter(User.id == user_id).first()
    return user

# mypy が警告
result = get_user_by_id("abc")  # error: Argument 1 has incompatible type "str"; expected "int"
```

### 型ヒントの基本

```python
from typing import List, Dict, Optional, Union

# 基本型
def greet(name: str) -> str:
    return f"Hello, {name}"

# Optional（None の可能性）
def get_user(user_id: int) -> Optional[User]:
    return db.query(User).filter(User.id == user_id).first()

# Union（複数の型の可能性）
def process_id(id: Union[int, str]) -> str:
    return str(id)

# Python 3.10+ の新しい記法
def get_user(user_id: int) -> User | None:  # Optional[User] と同じ
    ...

# List, Dict
def get_all_users() -> list[User]:
    return db.query(User).all()

def get_user_dict(user_id: int) -> dict[str, Any]:
    ...
```

### mypy の設定

```ini
# mypy.ini または pyproject.toml
[mypy]
python_version = 3.11
warn_return_any = True
warn_unused_configs = True
disallow_untyped_defs = True  # 型ヒントのない関数を禁止
ignore_missing_imports = True  # サードパーティライブラリの型エラーを無視

# 特定のモジュールを除外
[mypy-tests.*]
ignore_errors = True
```

### よくある型ヒントパターン

```python
from sqlalchemy.orm import Session
from fastapi import Depends

# サービス層
class UserService:
    def __init__(self, db: Session) -> None:
        self.db = db

    def create_user(self, user_data: UserCreate) -> User:
        ...

    def list_users(self, limit: int = 100) -> list[User]:
        ...

# ルートハンドラ
@app.post("/users", response_model=UserResponse)
def create_user(
    user: UserCreate,
    db: Session = Depends(get_db)
) -> User:  # 戻り値の型
    service = UserService(db)
    return service.create_user(user)

# Pydantic モデル（自動的に型付けされる）
class UserCreate(BaseModel):
    name: str
    email: str
```

### mypy の実行

```bash
# 全ファイルをチェック
mypy .

# 特定のファイルをチェック
mypy main.py services/

# エラーがあると終了コード 1
mypy . && echo "Type check passed!"
```

## Mission（本番課題）

### 状況

TL から以下の指示が来ました:

> 「型ヒントがないコードが多くて、IDE の補完が効かない。全関数に型ヒントを追加して、mypy を通してください。CI/CD にも mypy を組み込みたいので、設定ファイルも作って。」

### やること

1. **GitHub Issue を作成する**
   - タイトル: `refactor: 全関数に型ヒントを追加し、mypy を導入`

2. **ブランチを作成する**
   ```bash
   git checkout -b refactor/proj04-type-hints-mypy
   ```

3. **mypy をインストールする**
   ```bash
   pip install mypy
   pip freeze > requirements.txt
   ```

4. **mypy 設定ファイルを作成する**
   - ファイル: `mypy.ini` または `pyproject.toml`
   - 設定:
     - `python_version = 3.11`
     - `disallow_untyped_defs = True`
     - `ignore_missing_imports = True`

5. **全関数に型ヒントを追加する**
   - `main.py`: 全ルートハンドラ
   - `services/*.py`: 全サービスメソッド
   - `utils/*.py`: ユーティリティ関数

6. **mypy を実行し、エラーを修正する**
   ```bash
   mypy .
   ```
   - エラーが出る箇所を順番に修正
   - 難しい箇所は `# type: ignore` で一時的に無視

7. **mypy が通ることを確認する**
   ```bash
   mypy .
   # Success: no issues found
   ```

8. **CI/CD に mypy を追加する**
   - `.github/workflows/test.yml` に追加（GitHub Actions の場合）

9. **全テストが通ることを確認する**
   ```bash
   pytest tests/ -v
   ```

10. **PR を作成する**

<details>
<summary>型ヒント追加の例</summary>

```python
# Before
def create_user(user, db):
    service = UserService(db)
    return service.create_user(user)

# After
from sqlalchemy.orm import Session
from schemas import UserCreate, UserResponse

@app.post("/users", response_model=UserResponse)
def create_user(
    user: UserCreate,
    db: Session = Depends(get_db)
) -> User:
    service = UserService(db)
    return service.create_user(user)

# サービス層
class UserService:
    def __init__(self, db: Session) -> None:
        self.db = db

    def create_user(self, user_data: UserCreate) -> User:
        db_user = User(**user_data.dict())
        self.db.add(db_user)
        self.db.commit()
        self.db.refresh(db_user)
        return db_user

    def list_users(self) -> list[User]:
        return self.db.query(User).all()
```

</details>

<details>
<summary>よくある mypy エラーと修正</summary>

```python
# エラー 1: Function is missing a return type annotation
def get_user(user_id):  # error
    ...

def get_user(user_id: int) -> User | None:  # OK
    ...

# エラー 2: Missing type parameters for generic type
def list_users() -> list:  # error: Missing type parameters
    ...

def list_users() -> list[User]:  # OK
    ...

# エラー 3: Argument has incompatible type
def process_user(user: User):
    ...

process_user("invalid")  # error: Argument 1 has incompatible type "str"; expected "User"

# エラー 4: Incompatible return value type
def get_user_name(user_id: int) -> str:
    user = get_user(user_id)  # User | None
    return user.name  # error: Item "None" has no attribute "name"

# 修正
def get_user_name(user_id: int) -> str | None:
    user = get_user(user_id)
    return user.name if user else None
```

</details>

### 加点項目

- [ ] CI/CD に mypy が組み込まれている
- [ ] disallow_untyped_defs = True が設定されている
- [ ] 型ヒントの品質が高い（Any を使わない）
- [ ] docstring と型ヒントが一致している

### よくある間違い

- 型ヒントを一部だけ追加する（mypy が警告する関数が残る）
- `# type: ignore` を乱用する（型チェックの意味がない）
- Any を多用する（型安全性が低い）
- mypy の設定が甘い（warn_return_any などを無効にする）
- テストファイルにも型ヒントを強制する（ignore_errors = False）
